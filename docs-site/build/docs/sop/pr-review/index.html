PR Review 流程

代码审查流程和标准。

Review 流程​

Slaver 提交 PR
       ↓
Master 收到通知
       ↓
Master 审查代码
       ↓
   ┌───┴───┐
   ↓       ↓
 通过     需要修改
   ↓       ↓
 合并     反馈问题
         ↓
    Slaver 修复
         ↓
    重新审查

审查清单​

代码质量​

  • 代码清晰易懂
  • 遵循命名规范
  • 无重复代码
  • 函数长度合理

功能正确性​

  • 实现所有需求
  • 边界条件处理
  • 错误处理完善

测试覆盖​

  • 单元测试通过
  • 集成测试通过
  • 覆盖率达标

文档​

  • 更新相关文档
  • 添加必要的注释
  • 更新 CHANGELOG

Review 响应​

通过​

✅ LGTM (Looks Good To Me)
- 代码质量高
- 测试覆盖完整
- 可以直接合并

需要小修改​

需要修改：
1. 第 23 行：变量命名不清晰
2. 添加错误处理
3. 补充单元测试

修改后可以直接合并。

需要大改​

❌ 需要重新设计

问题：
1. 架构设计不合理
2. 缺少关键功能

建议：
1. 重新设计方案
2. 讨论后再实现
