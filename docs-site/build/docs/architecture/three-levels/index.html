三级架构详解

EKET 采用渐进式三级架构设计，确保在不同环境下都能稳定运行。

功能对比矩阵​

功能Level 1Level 2Level 3
Master-Slaver 协作✅✅✅
任务分配和认领✅✅✅
心跳监控✅✅✅
消息传递文件队列优化文件队列Redis Pub/Sub
消息持久化✅✅✅
消息去重❌✅✅
断路器❌✅✅
重试机制❌✅✅
LRU 缓存❌内存内存 + Redis
Master 选举文件锁文件锁Redis + SQLite + 文件
知识库❌❌✅
Web Dashboard❌✅✅

Level 1: Shell + 文档 (基础版) ⭐⭐⭐⭐⭐​

目标：所有核心功能可用，零配置启动

快速启动 (30 秒)​

# 1. 克隆仓库
git clone https://github.com/godlockin/eket.git
cd eket

# 2. 启动 Master (Shell 模式)
./scripts/eket-start.sh --role master

# 3. 启动 Slaver (Shell 模式)
./scripts/eket-start.sh --role slaver --profile backend_dev

# 完成！Level 1 已就绪

依赖要求​

  • ✅ Bash >= 4.0
  • ✅ Git >= 2.30
  • ❌ 不需要 Node.js
  • ❌ 不需要 Redis
  • ❌ 不需要任何安装

核心脚本​

脚本功能优先级
scripts/eket-start.sh启动 Master/SlaverP0
scripts/heartbeat-monitor.sh心跳监控P0
lib/adapters/hybrid-adapter.sh命令路由和降级P0

文件队列机制​

消息格式 (JSON):

{
  "id": "msg_abc123",
  "from": "slaver-001",
  "to": "master",
  "type": "task_complete",
  "timestamp": 1234567890,
  "payload": {
    "task_id": "TASK-001",
    "status": "done"
  }
}

Level 2: Node.js + 文件队列 (增强版) ⭐⭐⭐⭐​

目标：更高效、更专业、更丰富

快速启动​

# 1. 构建 Node.js 版本
cd node
npm install
npm run build
cd ..

# 2. 启动 Master (Node.js 模式)
node node/dist/index.js instance:start --role master

依赖要求​

  • ✅ Node.js >= 18.0.0
  • ✅ npm >= 9.0.0

相比 Level 1 增加​

  • ✅ TypeScript 类型安全
  • ✅ 丰富的 CLI 命令
  • ✅ 优化的文件队列（去重、归档、校验和）
  • ✅ 断路器和重试机制
  • ✅ LRU 内存缓存
  • ✅ Web Dashboard

CLI 命令​

# 系统诊断
node dist/index.js system:doctor

# 实例管理
node dist/index.js instance:start --auto

# Web 服务
node dist/index.js web:dashboard --port 3000

Level 3: Node.js + Redis + SQLite (满血版) ⭐⭐⭐​

目标：生产级、高并发、持久化

快速启动​

# 1. 启动 Redis
docker run -d --name eket-redis -p 6379:6379 redis:7-alpine

# 2. 启动 Master (满血模式)
node node/dist/index.js instance:start --role master

依赖要求​

  • ✅ Node.js >= 18.0.0
  • ✅ Redis >= 6.0.0
  • ✅ SQLite >= 3.35.0

相比 Level 2 增加​

  • ✅ Redis Pub/Sub 实时消息
  • ✅ SQLite 持久化存储
  • ✅ 分布式部署
  • ✅ 完整的监控和告警
  • ✅ 知识库系统

Redis 功能​

消息队列 (Pub/Sub):

const queue = new MessageQueue({
  useRedis: true,
  fallbackToFile: true
});

await queue.publish('tasks', message);

缓存层 (LRU + Redis):

const cache = new CacheLayer({
  useRedis: true,
  maxMemorySize: 100 * 1024 * 1024
});

await cache.set('key', value, 300);

降级决策树​

启动 EKET
    ↓
Node.js 可用？
    ├─ 否 → Level 1 (Shell)
    └─ 是 → Redis 可用？
            ├─ 否 → Level 2 (Node.js + 文件)
            └─ 是 → SQLite 可用？
                    ├─ 否 → Level 2
                    └─ 是 → Level 3 (满血版)

使用建议​

场景推荐级别原因
快速试用Level 1零配置，30 秒启动
本地开发Level 2丰富功能，无需外部依赖
团队协作Level 3实时消息，分布式支持
生产环境Level 3高可用，完整监控
