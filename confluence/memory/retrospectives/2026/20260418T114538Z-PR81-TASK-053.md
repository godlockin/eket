## What Hurt + Root Cause
- **Copy-paste 引入 corrupted step**：PR #77 合并时 `Enable auto-merge` step 被复制两份，第一份 run block 混入 YAML-as-bash 残片（`truer` 拼写错误）
  - 根因：手动编辑 workflow 文件，复制粘贴未做 diff 审查
  - 侥幸可用原因：`|| true` 吞掉了 garbage 命令的非零退出码，运行时无报错
  - 发现时机：AI-Review 附件展示时才注意到 `truer` 疑点，属于事后发现
- **YAML lint 缺失**：CI 无 YAML 语法/语义 lint 步骤，corrupted step 得以通过 required checks

## 经验教训
- **Copy-paste 必须 diff**：复制 workflow step 后务必对比两份内容，或用 YAML linter 验证；`|| true` 会掩盖 bash 错误，不能依赖运行时报错发现问题
- **actionlint 价值**：GitHub Actions workflow 文件建议加 `actionlint` CI 检查，能在 merge 前捕获 YAML 结构错误和非法 bash 片段
- **AI-Review 附件展示有盲区**：AI review 在正文 diff 视图中未能直接发现重复 step，附件/全文视图才触发发现——说明复杂 workflow 变更需要额外的全文审查
