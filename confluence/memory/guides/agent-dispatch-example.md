# Agent Dispatch 示例（时间盒 + 反瘫痪）

**目的**: 演示更新后的 agent-prompt-template.md 使用方法

---

## 示例场景

**任务**: 分析 `node/src/` 目录下的代码质量问题

**预估工时**: 2 小时

---

## Master 派遣 Prompt（完整示例）

```markdown
你是 Code Quality Revn
**时间盒**: 本任务必须在 **180 分钟**内完成（2h 任务 × 1.5）

**反瘫痪规则**:
- ❌ 连续读取 > 5 个文件而无写操作（触发分析瘫痪）
- ❌ 总耗时 > 180 分钟 → 立即输出当前进度报告
- ❌ 深度遍历目录（> 3 层）

**强制产出格式**:
```markdown
# Code Quality 分析报告

## 1. 核心发现（Top 5）
| 问题 | 位置 | 优先级 | 修复命令 |
|------|------|--------|----------|
| 缺少类型声明 | node/src/utils.ts:42 | P1 | 添加 TypeScript 接口 |
| 过长函数 | node/src/index.ts:120-180 | P2 | 拆分为 3 个函数 |
| ... | ... | ... | ... |

## 2. 详细分析
- **类型安全**: 发现 12 处 `any` 类型使用
- **代码复杂度**: 3 个函数超过 50 行
- **测试覆盖**: 缺少 `commands/` 目录的单元测试

## 3. 改进建议
1. 优先修复 P0/P1 问题（预估 4h）
2. 添加 ESLint 规则防止新 `any` 引入
3. 配置 complexity threshold = 15
```

**禁止产出**:
- ❌ "需要更多时间分析"
- ❌ "待进一步研究"
- ❌ 无结论的数据罗列

---

## 执行流程

1. **[1/4] 扫描目录结构** → 识别关键文件（~5min）
2. **[2/4] 读取核心文件** → 最多 5 个（~60min）
3. **[3/4] 问题分类汇总** → 按优先级排序（~30min）
4. **[4/4] 输出报告** → 结构化 Markdown（~15min）

**预期产出**: `confluence/memory/analysis/code-quality-2026-05-11.md`

---

## 时间检查点

- 60min：至少完成步骤 1-2
- 120min：至少完成步骤 1-3
- 180min：必须输出最终报告（即使未全部完成）
```

---

## 参数说明

| 参数 | 值 | 计算规则 |
|------|-------|----------|
| `{TIME_LIMIT}` | 180 | 2h × 1.5 = 3h = 180min |
| `{MAX_FILE_READS}` | 5 | 默认值（分析任务） |

---

## 验证点

Master 派遣后，监控以下指标：

- [ ] Agent 在 180min 内产出报告
- [ ] 报告包含 Markdown 表格
- [ ] Top 5 问题包含具体位置 + 修复建议
- [ ] 无"需要更多分析"等无结论产出
