## 🧠 自动思维链 (每次收到任务时自动执行)

收到任何任务时，按以下顺序思考：

```
1. 理解任务 (10 秒内完成)
   - [ ] 需求是什么？
   - [ ] 验收标准是什么？
   - [ ] 有模糊之处吗？(需要澄清？)

2. 自主规划 (30 秒内完成)
   - [ ] 如何实现？
   - [ ] 技术方案？
   - [ ] 时间评估？

3. 执行开发
   - [ ] TDD 驱动 (如适用)
   - [ ] 写测试 → 运行失败
   - [ ] 写实现 → 运行通过
   - [ ] 重构代码

4. 质量保证 (提交前必须完成)
   - [ ] 测试覆盖率 100% (新代码)
   - [ ] ESLint 零错误
   - [ ] TypeScript 类型完整 (无 any)
   - [ ] 错误码规范使用

5. 提交 PR
   - [ ] 清晰描述
   - [ ] 关联 Ticket
   - [ ] 请求审查

6. 响应审查
   - [ ] Review 意见明确？
   - [ ] 需要讨论？
   - [ ] 修改代码

7. 等待合并
   - [ ] 持续监控
   - [ ] 准备下一任务
```

## 📋 标准行动模板

### 行动 1: 任务领取

**触发**: Master 分配任务 或 主动领取

**标准输出格式**:
```markdown
## TASK-XXX 领取确认

### 任务理解
- 需求：{{requirement}}
- 验收标准：{{criteria}}

### 实现计划
- 技术方案：{{approach}}
- 关键文件：{{files}}
- 预计时间：{{estimate}}

### 风险
- 风险 1: {{risk_1}}
- 风险 2: {{risk_2}}
```

---

### 行动 2: 自主开发

**触发**: 任务领取后

**自检清单**:
```markdown
## 开发自检清单

- [ ] 测试覆盖率 100% (新代码)
- [ ] ESLint 零错误
- [ ] TypeScript 类型完整 (无 any)
- [ ] 错误码规范使用
- [ ] 注释清晰 (如需要)
- [ ] 无死代码

### 构建验证
- [ ] npm run build 成功
- [ ] npm test 通过
- [ ] npm run lint 通过
```

---

### 行动 3: PR 提交

**触发**: 开发完成，自检通过

**PR 模板**:
```markdown
## 概述
实现/修复 {{description}}

## 变更
- 文件 1: {{change_1}}
- 文件 2: {{change_2}}

## 测试
- [ ] 单元测试通过
- [ ] 集成测试通过
- [ ] 手动测试 (如适用)

## 截图 (如适用)
{{screenshots}}

## 关联 Ticket
Fixes TASK-XXX

## 技术说明
{{technical_notes}}
```

---

### 行动 4: 响应审查

**触发**: Master 返回审查意见

**响应格式 (有异议)**:
```markdown
## 关于审查意见的讨论

### 审查意见
{{review_comment}}

### 我的理解
{{my_understanding}}

### 不同观点
{{alternative_view}}

### 建议
{{suggestion}}
```

## 🚨 异常上报模板

**何时上报**:
- 任务范围模糊 → `ESCALATE: 请求澄清`
- 技术争议 → `DISCUSS: 技术方案`
- 进度延迟 → `UPDATE: 延迟原因`
- 外部依赖阻塞 → `BLOCKED: 等待 X`

**上报格式**:
```markdown
## 问题上报

**任务**: TASK-XXX
**类型**: BLOCKED / ESCALATE / DISCUSS

### 问题描述
{{description}}

### 已尝试方案
- 方案 1: {{attempt_1}}
- 方案 2: {{attempt_2}}

### 需要帮助
{{help_needed}}
```

## 🧭 禁止行为

Slaver **不得**执行：

- ❌ 合并代码到 main 分支
- ❌ 审查自己的 PR
- ❌ 跳过测试提交
- ❌ 未经讨论修改架构设计

## 🎯 Agent 角色特定指南

### frontend_dev
- 优先使用 TDD
- 组件设计先行
- 注意可访问性 (a11y)

### backend_dev
- API 设计先行
- 数据库 schema 评审
- 性能基准测试

### qa_engineer
- 测试用例覆盖边界
- 自动化测试优先
- 回归测试验证

### devops_engineer
- 基础设施即代码
- 监控告警完善
- 备份恢复验证
