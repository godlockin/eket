## 代码分析命令

### eket analyze:structure

提取代码结构（函数/类/导入/导出），不调用 LLM。

```bash
eket analyze:structure <path> [options]
```

**Options:**
- `--output <file>` — 输出 JSON 文件路径
- `--language <lang>` — 指定语言（自动检测时使用）

**Example:**
```bash
eket analyze:structure src/core/election.rs
eket analyze:structure src/ --output .eket/structure.json
```

### eket batch:compute

计算文件批次，用于大任务拆解。

```bash
eket batch:compute <path> [options]
```

**Options:**
- `--max-nodes <n>` — 每批最大节点数（默认 60）
- `--output <file>` — 输出 JSON 文件路径

**Example:**
```bash
eket batch:compute src/ --max-nodes 40 --output .eket/batches.json
```

### eket fingerprint:build / diff

管理文件指纹，支持增量分析。

```bash
eket fingerprint:build <path> --output <file>
eket fingerprint:diff --baseline <file>
```

**Example:**
```bash
eket fingerprint:build src/ --output .eket/fingerprints.json
eket fingerprint:diff --baseline .eket/fingerprints.json
```
```

## 确定性脚本 + LLM 分离

EKET 采用混合架构处理代码分析：

```
┌─────────────────────────────────────────────┐
│           确定性层 (Rust)                    │
├─────────────────────────────────────────────┤
│ • tree-sitter AST 解析                       │
│ • 导入关系提取                               │
│ • 批次分割 + neighborMap                     │
│ • fingerprint 计算                     ───────────────────────────────────────────┐
│           语义层 (LLM)                       │
├─────────────────────────────────────────────┤
│ • summary 生成                              │
│ • tags 标注                                 │
│ • complexity 评估                           │
│ • 架构层识别                                │
└─────────────────────────────────────────────┘
```

**设计原则：**
1. 确定性操作不用 LLM（文件列表、导入解析、AST 提取）
2. LLM 只处理语义（总结、标签、复杂度）
3. 中间结果用 JSON 交换，支持断点续跑
```

### 3. 使用指南

创建 `docs/guides/incremental-analysis.md`：

```markdown
# 增量分析指南

## 概述

增量分析只重新分析变更的文件，节省时间和资源。

## 工作流

### 1. 建立基线

首次分析时建立 fingerprint 基线：

```bash
eket analyze:structure src/ --output .eket/structure.json
eket fingerprint:build src/ --output .eket/fingerprints.json
```

### 2. 检测变更

开发后检测变更类型：

```bash
eket fingerprint:diff --baseline .eket/fingerprints.json
```

输出示例：
```
STRUCTURAL: 3  (需要重分析)
COSMETIC: 12   (只改了注释/空格，跳过)
NEW: 1         (新文件)
DELETED: 0
```

### 3. 增量分析

只分析 STRUCTURAL + NEW 文件：

```bash
eket analyze:incremental --baseline .eket/fingerprints.json
```

## 变更分类

| 类型 | 含义 | 处理 |
|------|------|------|
| STRUCTURAL | 函数签名/类结构变化 | 重新分析 |
| COSMETIC | 只改注释/空格 | 跳过 |
| NEW | 新文件 | 分析 |
| DELETED | 删除的文件 | 从索引移除 |
```

### 代码节点
- `file` — 源代码文件

- `config` — 配置文件 (yaml/json/toml)

- `pipeline` — CI/CD 配置
- `schema` — Schema 定义 (GraphQL/Proto)

## 边类型

### 非代码边
- `configures` — 配置文件 → 代码
- `documents` — 文档 → 代码
- `deploys` — 容器 → 代码
- `triggers` — CI/CD → 服务
- `defines_schema` — Schema → 代码

#### 内容策略视角

从内容角度审视本任务，识别三个关键特征：

1. **功能链路闭环**: EPIC-011 引入 4 个核心能力（tree-sitter 结构提取、批次分割、fingerprint 增量、非代码 schema），文档需完整覆盖"是什么 → 为什么 → 怎么用"全链路
