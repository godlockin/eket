## 目标

用 SWE-bench Lite（500 个验证过的 issue）评测 EKET Slaver 的代码修复能力。

## 评测方法

### 1. 评测维度

| 维度 | 指标 | 说明 |
|------|------|------|
| 准确率 | Pass@1 | 首次生成 patch 修复成功率 |
| 效率 | Tokens/Issue | 平均每个 issue 消耗 token |
| 成本 | $/Issue | 平均每个 issue 成本 |
| 时间 | s/Issue | 平均每个 issue 耗时 |

### 2. 对比基线

- GPT-4 直接调用（无 Agent）
- mini-swe-agent（100 行极简）
- EKET Slaver（完整框架）

### 3. 评测流程

```
1. 选取 SWE-bench Lite 样本（10-50 个 issue）
2. 对每个 issue：
   a. 提供 issue 描述 + 代码库上下文
   b. 调用 EKET Slaver 生成 patch
   c. 验证 patch 是否通过测试
   d. 记录 token/时间/成本
3. 生成评测报告
```
