## 背景

EKET 有多个 skill（eket、dynamic-expert-group 等），但缺乏跨 skill 的统一行为约束。Karpathy 的 4 条原则适用于所有编码场景。

## 目标

创建 `META-GUIDELINES.md` 作为所有 EKET skill 的顶层行为约束。

## 技术方案

### 1. 文件位置

```
.claude/skills/eket/META-GUIDELINES.md
```

### 2. 内容结构

```markdown
# EKET 元规则

> 所有 EKET skill 必须遵守的顶层行为约束

## 四大原则

### 1. 🧠 Think Before Coding
- 明确列出假设
- 有多种解读时询问而非猜测
- 存在更简单方案时主动提出

### 2. ✂️ Simplicity First
- 只写解决问题所需的最小代码
- 不添加未请求的功能
- 不为假设场景预设抽象

### 3. 🔬 Surgical Changes
- 只改必须改的
- 保持现有代码风格
- 只清理自己引入的死代码

### 4. 🎯 Goal-Driven Execution
- 将任务转化为可验证的成功标准
- 多步计划包含验证检查点
- 先写测试再实现

## 应用方式

每个 skill 的 SKILL.md 应在开头引用：
```markdown
> 📋 本 skill 遵循 [META-GUIDELINES.md](../META-GUIDELINES.md)
```

## 豁免场景

- 紧急 hotfix 可跳过完整流程
- 用户明确要求时可违反原则
- 探索性原型开发可放宽限制
```

### 3. 引用机制

更新现有 skill 添加元规则引用。
