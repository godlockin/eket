# TASK-E12-001: Gate Review 增加 Surgical Changes 检查

## 元信息

| 字段 | 值 |
|------|-----|
| Epic | EPIC-012 |
| 优先级 | P0 |
| 估时 | 2h |
| 状态 | `done` |
| 依赖 | 无 |

## 背景

Karpathy Guidelines 的 "Surgical Changes" 原则：
> "Touch only what you must. Clean up only your own mess."

当前 EKET gate:review 缺乏对改动范围的检查，容易出现：
- 修 bug 时顺手重构无关代码
- 添加功能时改变既有风格
- diff 过大导致 review 困难

## 目标

在 gate:review 中增加 Surgical Changes 检查，警告过大或过散的改动。

## 技术方案

### 1. 新增检查项

```rust
// rust/crates/eket-core/src/gate/checks/surgical.rs

pub struct SurgicalChangesCheck;

#[derive(Debug)]
pub struct SurgicalReport {
    pub files_changed: usize,
    pub lines_added: usize,
    pub lines_removed: usize,
    pub unrelated_changes: Vec<UnrelatedChange>,
    pub severity: Severity,
}

#[derive(Debug)]
pub struct UnrelatedChange {
    pub file: String,
    pub reason: String, // e.g., "style-only change", "unrelated refactor"
}

impl SurgicalChangesCheck {
    pub fn analyze(diff: &str, task_scope: &TaskScope) -> SurgicalReport {
        // 1. 解析 diff 统计
        // 2. 检测与任务范围不符的文件
        // 3. 检测纯风格变更
        // 4. 计算严重度
    }
}
```

### 2. 阈值配置

```toml
# .eket/config.toml
[gate.surgical]
max_files = 10           # 单次改动最大文件数
max_lines = 500          # 单次改动最大行数
warn_style_only = true   # 警告纯风格变更
```

### 3. 输出格式

```
🔬 Surgical Changes Check
├── Files: 3 (≤10 ✓)
├── Lines: +45 / -12 (≤500 ✓)
├── Scope match: 2/3 files in task scope
└── Warnings:
    ⚠ src/utils/format.rs — style-only change (not in task scope)
```

## 验收标准

- [ ] `eket gate:review` 输出包含 Surgical Changes 检查
- [ ] 超过阈值时显示警告
- [ ] 检测纯风格变更（仅空格/注释/格式）
- [ ] 阈值可配置
- [ ] 单元测试覆盖

## 测试用例

```rust
#[test]
fn test_detect_style_only_change() {
    let diff = r#"
-    let x = 1;
+    let x=1;
"#;
    let report = SurgicalChangesCheck::analyze(diff, &TaskScope::default());
    assert!(report.unrelated_changes.iter().any(|c| c.reason == "style-only change"));
}
```
