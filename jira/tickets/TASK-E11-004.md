# TASK-E11-004: 扩展 ticket schema 支持非代码文件

## 元信息

## 背景

Understand-Anything 的 schema 设计了 21 种节点类型，其中 8 种专门用于非代码文件：

| 类型 | 用途 | ID 格式 |
|------|------|---------|
| `config` | 配置文件 (yaml/json/toml) | `config:<path>` |
| `document` | 文档 (md/rst) | `document:<path>` |
| `service` | 容器定义 (Dockerfile/K8s) | `service:<path>` |
| `table` | 数据库表 (SQL) | `table:<path>:<name>` |
| `endpoint` | API 端点 (OpenAPI) | `endpoint:<path>:<name>` |
| `pipeline` | CI/CD 配置 | `pipeline:<path>` |
| `schema` | Schema 定义 (GraphQL/Proto) | `schema:<path>` |
| `resource` | 基础设施资源 (Terraform) | `resource:<path>` |

这让非代码文件也能参与知识图谱分析。

## 目标

扩展 EKET 的 ticket schema，支持非代码文件的分类和分析。

## 技术方案

### 1. 扩展 FileCategory

```rust
// rust/src/types/file.rs

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
#[serde(rename_all = "lowercase")]
pub enum FileCategory {
    Code,    // 源代码
    Config,  // 配置文件
    Docs,    // 文档
    Infra,   // 基础设施
    Data,    // 数据/Schema
    Script,  // 脚本
    Markup,  // HTML/CSS
}

impl FileCategory {
    pub fn from_path(path: &Path) -> Self {
        let filename = path.file_name().and_then(|s| s.to_str()).unwrap_or("");
        let ext = path.extension().and_then(|s| s.to_str()).unwrap_or("");
        
        // 优先匹配文件名
        match filename {
            "Dockerfile" | "Jenkinsfile" | "Makefile" | "Vagrantfile" => return FileCategory::Infra,
            "LICENSE" => return FileCategory::Code,
            _ => {}
        }
        
        // 路径匹配
        let path_str = path.to_string_lossy();
        if path_str.contains(".github/workflows") || path_str.contains(".gitlab-ci") {
            return FileCategory::Infra;
        }
        if path_str.contains("k8s/") || path_str.contains("kubernetes/") {
            return FileCategory::Infra;
        }
        
        // 扩展名匹配
        match ext {
            // Config
            "yaml" | "yml" | "json" | "toml" | "ini" | "env" | "cfg" => FileCategory::Config,
            // Docs
            "md" | "mdx" | "rst" | "txt" => FileCategory::Docs,
            // Infra
            "tf" | "tfvars" => FileCategory::Infra,
            // Data
            "sql" | "graphql" | "gql" | "proto" | "prisma" => FileCategory::Data,
            // Script
            "sh" | "bash" | "zsh" | "ps1" | "bat" | "cmd" => FileCategory::Script,
            // Markup
            "html" | "htm" | "css" | "scss" | "sass" | "less" => FileCategory::Markup,
            // Default
            _ => FileCategory::Code,
        }
    }
}
```

### 2. 扩展 NodeType

```rust
// rust/src/types/node.rs

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
#[serde(rename_all = "lowercase")]
pub enum NodeType {
    // Code nodes
    File,
    Function,
    Class,
    Module,
    
    // Non-code nodes
    Config,
    Document,
    Service,
    Table,
    Endpoint,
    Pipeline,
    Schema,
    Resource,
}

impl NodeType {
    pub fn from_category_and_content(category: FileCategory, path: &Path, content: &str) -> Self {
        match category {
            FileCategory::Config => NodeType::Config,
            FileCategory::Docs => NodeType::Document,
            FileCategory::Infra => {
                let filename = path.file_name().and_then(|s| s.to_str()).unwrap_or("");
                if filename.starts_with("Dockerfile") || filename.contains("docker-compose") {
                    NodeType::Service
                } else if path.to_string_lossy().contains(".github/workflows") {
                    NodeType::Pipeline
                } else {
                    NodeType::Resource
                }
            }
            FileCategory::Data => {
                let ext = path.extension().and_then(|s| s.to_str()).unwrap_or("");
                if ext == "sql" && content.contains("CREATE TABLE") {
                    NodeType::Table
                } else if ext == "graphql" || ext == "proto" || ext == "prisma" {
                    NodeType::Schema
                } else {
                    NodeType::Schema
                }
            }
            _ => NodeType::File,
        }
    }
}
```

### 4. 非代码文件分析

```rust
// rust/src/analyzer/non_code.rs

pub fn analyze_non_code_file(path: &Path, category: FileCategory) -> NonCodeAnalysis {
    let content = std::fs::read_to_string(path).unwrap_or_default();
    
    match category {
        FileCategory::Config => analyze_config(&content, path),
        FileCategory::Docs => analyze_docs(&content, path),
        FileCategory::Infra => analyze_infra(&content, path),
        FileCategory::Data => analyze_data(&content, path),
        _ => NonCodeAnalysis::default(),
    }
}

fn analyze_config(content: &str, path: &Path) -> NonCodeAnalysis {
    let ext = path.extension().and_then(|s| s.to_str()).unwrap_or("");
    match ext {
        "yaml" | "yml" => analyze_yaml(content),
        "json" => analyze_json(content),
        "toml" => analyze_toml(content),
        _ => NonCodeAnalysis::default(),
    }
}

fn analyze_infra(content: &str, path: &Path) -> NonCodeAnalysis {
    let filename = path.file_name().and_then(|s| s.to_str()).unwrap_or("");
    if filename.starts_with("Dockerfile") {
        analyze_dockerfile(content)
    } else if filename.contains("docker-compose") {
        analyze_docker_compose(content)
    } else if path.to_string_lossy().contains(".github/workflows") {
        analyze_github_workflow(content)
    } else {
        NonCodeAnalysis::default()
    }
}
```

## 专家分析

### 老码 (CodeMaster Wang) — 代码质量视角

#### 类型设计审查

**FileCategory 问题**

1. **大小写敏感** — `Dockerfile` 匹配是 case-sensitive，但实际项目中存在 `dockerfile`、`DOCKERFILE` 变体
   ```rust
   // 修复：使用 eq_ignore_ascii_case
   if filename.eq_ignore_ascii_case("Dockerfile") { ... }
   ```

2. **`.env.example` 误分类** — `extension()` 返回 `example` 而非 `env`
   ```rust
   // 修复：文件名前缀匹配
   if filename.starts_with(".env") { return FileCategory::Config; }
   ```

3. **无扩展名配置文件** — `.eslintrc`、`.prettierrc` 等没有扩展名
   ```rust
   // 修复：增加 dotfile 白名单
   const DOTFILE_CONFIGS: &[&str] = &[".eslintrc", ".prettierrc", ".gitignore", ...];
   ```

4. **LICENSE 分类不当** — 应为 `Meta` 类型而非 `Code`

**NodeType 问题**

1. **SQL 注释误判** — `content.contains("CREATE TABLE")` 会匹配注释中的文本
   ```rust
   // 修复：使用正则排除注释
   let re = Regex::new(r"(?m)^\s*CREATE\s+TABLE").unwrap();
   ```

2. **穷尽性不足** — `Script` 和 `Markup` 类别直接 fallback 到 `File`，缺乏语义

3. **Endpoint 映射缺失** — ticket 定义了 `NodeType::Endpoint` 但 `from_category_and_content` 没有映射逻辑

**EdgeType 设计**

- 当前设计合理，但考虑是否需要反向边类型（`Configures` ↔ `ConfiguredBy`）

#### 边界情况清单

| 场景 | 当前行为 | 建议 |
|------|----------|------|
| 空路径 `""` | `file_name()` 返回 `None` → fallback Code | 显式处理 |
| 多扩展名 `schema.graphql.bak` | 分类为 Code | 考虑二级扩展名 |
| 符号链接 | 基于链接名分类 | 文档化行为 |
| Windows 路径 `k8s\deployment.yaml` | `contains("k8s/")` 不匹配 | 使用 Path API |
| 隐藏目录 `.github/workflows/ci.yml` | 正确识别为 Infra | ✓ |

#### 风险与缓解

| 风险 | 严重度 | 缓解 |
|------|--------|------|
| 大小写导致 Dockerfile 漏检 | **高** | `eq_ignore_ascii_case()` |
| `.env.example` 错误分类 | 中 | 文件名前缀匹配 |
| SQL 注释误判 | 中 | 正则排除行首注释 |
| 无扩展名 dotfile 漏检 | 中 | 常见 dotfile 白名单 |
| `unwrap_or_default()` 隐藏错误 | 低 | 添加 tracing 日志 |

#### 代码修改建议

```rust
impl FileCategory {
    pub fn from_path(path: &Path) -> Self {
        let filename = path.file_name().and_then(|s| s.to_str()).unwrap_or("");
        let ext = path.extension().and_then(|s| s.to_str()).unwrap_or("");
        let path_str = path.to_string_lossy();

        // 1. 文件名精确匹配（case-insensitive）
        if filename.eq_ignore_ascii_case("Dockerfile")
            || filename.eq_ignore_ascii_case("Jenkinsfile")
            || filename.eq_ignore_ascii_case("Makefile")
            || filename.eq_ignore_ascii_case("Vagrantfile")
        {
            return FileCategory::Infra;
        }

        // 2. .env 系列
        if filename.starts_with(".env") {
            return FileCategory::Config;
        }

        // 3. 无扩展名 dotfile 配置
        const DOTFILE_CONFIGS: &[&str] = &[
            ".eslintrc", ".prettierrc", ".gitignore", ".editorconfig",
            ".dockerignore", ".npmrc", ".nvmrc", ".babelrc",
        ];
        if DOTFILE_CONFIGS.iter().any(|d| filename.eq_ignore_ascii_case(d)) {
            return FileCategory::Config;
        }

        // 4. 路径匹配（使用 Path API 处理跨平台）
        if path_str.contains(".github/workflows") || path_str.contains(".gitlab-ci") {
            return FileCategory::Infra;
        }
        if path.iter().any(|c| c == "k8s" || c == "kubernetes") {
            return FileCategory::Infra;
        }

        // 5. 扩展名匹配
        match ext.to_lowercase().as_str() {
            "yaml" | "yml" | "json" | "toml" | "ini" | "cfg" => FileCategory::Config,
            "md" | "mdx" | "rst" | "txt" => FileCategory::Docs,
            "tf" | "tfvars" => FileCategory::Infra,
            "sql" | "graphql" | "gql" | "proto" | "prisma" => FileCategory::Data,
            "sh" | "bash" | "zsh" | "ps1" | "bat" | "cmd" => FileCategory::Script,
            "html" | "htm" | "css" | "scss" | "sass" | "less" => FileCategory::Markup,
            _ => FileCategory::Code,
        }
    }
}
```
