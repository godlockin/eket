# TASK-E12-002: 专家 Persona 增加 Mantras 字段

## 元信息

| 字段 | 值 |
|------|-----|
| Epic | EPIC-012 |
| 优先级 | P1 |
| 估时 | 1.5h |
| 状态 | `done` |
| 依赖 | 无 |

## 背景

Karpathy Guidelines 使用口诀化表达增强记忆：
- "Don't assume. Don't hide confusion."
- "Minimum code. Nothing speculative."
- "Touch only what you must."

当前 EKET 专家 persona 缺乏类似的行为戒律字段。

## 目标

为专家 persona 增加 `mantras` 字段，在专家输出时引用相关口诀。

## 技术方案

### 1. 扩展专家 Schema

```yaml
# experts/default/backend.md 示例
id: eket.backend.001
name: Backend Expert
mantras:
  - "Type first, implement second."
  - "If it's not tested, it's broken."
  - "Fail fast, recover gracefully."
```

### 2. 输出模板调整

```markdown
## 🛡️ 老码的分析

> 💬 "Type first, implement second."

### 视角
...
```

### 3. 口诀库

为每个默认专家定义 2-3 条口诀：

| 专家 | 口诀 |
|------|------|
| 陈架构 | "Simplicity is the ultimate sophistication." / "Make it work, make it right, make it fast." |
| 老码 | "Type first, implement second." / "If it's not tested, it's broken." |
| 产品赵 | "User value over technical elegance." / "Ship, measure, iterate." |
| 内容孙 | "Show, don't tell." / "One idea per paragraph." |

## 验收标准

- [ ] 专家 YAML schema 支持 `mantras` 数组
- [ ] 动态专家组输出包含口诀引用
- [ ] 默认专家配置包含 2-3 条口诀
- [ ] 口诀与专家领域匹配

## 测试用例

```rust
#[test]
fn test_expert_mantras_loaded() {
    let expert = load_expert("eket.backend.001");
    assert!(!expert.mantras.is_empty());
    assert!(expert.mantras[0].len() < 100); // 口诀应简短
}
```
