## 背景

Understand-Anything 的核心设计是「确定性脚本 + LLM 分离」：
- **tree-sitter** 做代码结构提取（函数/类/导入/导出）
- **LLM** 只做语义增强（summary/tags/complexity）

这种分离的好处：
1. 结构提取 100% 可复现，不会幻觉
2. LLM 上下文更精简，只处理语义
3. 支持增量更新（fingerprint 基于 AST hash）

## 目标

在 EKET Rust CLI 中集成 tree-sitter，实现 `eket analyze:structure <path>` 命令。

## 技术方案

### 1. 依赖添加

```toml
# rust/Cargo.toml
[dependencies]
tree-sitter = "0.22"
tree-sitter-typescript = "0.22"
tree-sitter-javascript = "0.22"
tree-sitter-python = "0.22"
tree-sitter-rust = "0.22"
tree-sitter-go = "0.22"
```

### 2. 输出 Schema

```rust
#[derive(Serialize)]
pub struct StructuralAnalysis {
    pub path: String,
    pub language: String,
    pub functions: Vec<FunctionInfo>,
    pub classes: Vec<ClassInfo>,
    pub imports: Vec<ImportInfo>,
    pub exports: Vec<ExportInfo>,
    pub metrics: FileMetrics,
}

#[derive(Serialize)]
pub struct FunctionInfo {
    pub name: String,
    pub start_line: u32,
    pub end_line: u32,
    pub params: Vec<String>,
    pub return_type: Option<String>,
}

#[derive(Serialize)]
pub struct ClassInfo {
    pub name: String,
    pub start_line: u32,
    pub end_line: u32,
    pub methods: Vec<String>,
    pub properties: Vec<String>,
}

#[derive(Serialize)]
pub struct ImportInfo {
    pub source: String,
    pub specifiers: Vec<String>,
    pub line_number: u32,
}

#[derive(Serialize)]
pub struct ExportInfo {
    pub name: String,
    pub line_number: u32,
    pub is_default: bool,
}

#[derive(Serialize)]
pub struct FileMetrics {
    pub total_lines: u32,
    pub non_empty_lines: u32,
    pub import_count: u32,
    pub export_count: u32,
    pub function_count: u32,
    pub class_count: u32,
}
```

### 3. CLI 接口

```bash
# 单文件分析
eket analyze:structure src/main.rs

# 目录批量分析
eket analyze:structure src/ --output .eket/intermediate/structure.json

# 指定语言
eket analyze:structure src/utils.ts --language typescript
```

### 4. 语言检测

```rust
fn detect_language(path: &Path) -> Option<&'static str> {
    match path.extension()?.to_str()? {
        "ts" | "tsx" => Some("typescript"),
        "js" | "jsx" | "mjs" | "cjs" => Some("javascript"),
        "py" => Some("python"),
        "rs" => Some("rust"),
        "go" => Some("go"),
        "java" => Some("java"),
        "rb" => Some("ruby"),
        _ => None,
    }
}
```

## 专家分析

### 陈架构 (Alex Chen) — 系统架构评审

**评审日期**: 2026-05-24

#### 架构视角

1. **模块边界清晰**: tree-sitter 是纯解析库，无 I/O、无网络、无状态，天然适合作为 eket-core 子模块
2. **依赖链影响**: tree-sitter 0.22 + 5 语言 grammar 约增加 ~15-20MB 编译产物；语言 grammar 使用 C 代码编译，CI 需确认 C 编译器可用
3. **现有架构契合度**: eket-cli 已有 subcommand 模式，新增 `analyze:structure` 遵循现有范式

#### 细化模块划分

```
rust/crates/eket-core/src/
├── analyzer/              # 新增模块组
│   ├── mod.rs            # 统一 re-export
│   ├── tree_sitter.rs    # tree-sitter 核心封装 (Parser 管理, Query 执行)
│   ├── language.rs       # 语言检测 + grammar 延迟加载 (OnceLock)
│   ├── extractors/       # 各语言提取器
│   │   ├── mod.rs
│   │   ├── typescript.rs # TS/JS 共用（差异通过 query 区分）
│   │   ├── python.rs
│   │   ├── rust.rs
│   │   └── go.rs
│   └── types.rs          # StructuralAnalysis 等类型定义
```

#### 关键实现细节

**1. Query-based 提取** (非遍历式):
- 每种语言预定义 S-expression query（性能优于递归遍历）
- TypeScript 函数提取示例:
  ```scm
  (function_declaration
    name: (identifier) @fn.name
    parameters: (formal_parameters) @fn.params) @fn.def
  ```

**2. 语言 Grammar 延迟加载**:
```rust
use std::sync::OnceLock;
static TS_LANGUAGE: OnceLock<tree_sitter::Language> = OnceLock::new();

fn get_ts_language() -> &'static tree_sitter::Language {
    TS_LANGUAGE.get_or_init(|| tree_sitter_typescript::language_typescript())
}
```

**3. TS/JS 共用策略**:
- `tree-sitter-typescript` 包含 `language_typescript()` 和 `language_tsx()`
- JS 文件用 TypeScript parser 解析（兼容性 >99%）

#### 边界情况处理

| 场景 | 处理策略 |
|------|---------|
| 语法错误文件 | 返回 partial result + `errors: [{ line, message }]` |
| 空文件 | 合法空结构体 (`functions: [], classes: []`) |
| 超大文件 (>1MB) | 警告日志，继续处理 |
| 未知扩展名 | `{ "error": "unsupported_language", "path": "..." }` |
| 二进制文件 | 快速失败（检测前 8KB 是否为文本） |

#### 测试策略

| 测试类型 | 位置 | 覆盖范围 |
|---------|------|---------|
| 单元测试 | `eket-core/src/analyzer/*.rs` 内 `#[cfg(test)]` | 每个 extractor 函数 |

#### 风险与缓解

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| C 编译依赖 | 中 | Dockerfile/CI 添加 `build-essential`; README 明确系统要求 |
| 跨语言 Query 不一致 | 中 | 每语言 Query 独立定义，单元测试覆盖每个 capture |
| Grammar 版本锁定 | 低 | Cargo.toml 锁定 `=0.22.x`; 升级走独立 PR |
| 嵌套语言 (如 TSX) | 低 | 首版只支持单语言; TSX 用 tsx parser 独立处理 |
| 内存峰值 (大型 monorepo) | 低 | 批量模式添加 `--parallel` flag + rayon |
