## Common Rationalizations

> ⚠️ 非穷举清单 — LLM 可能用未列措辞绕过；Master 二次 review 是最后兜底。

| 借口 | 反驳 |
|------|------|
| "手测一下就够了，不用写自动化" | 手测一次省 5 分钟，每次回归省的是 0；自动化是把一次性投入摊销给所有未来回归 |
| "覆盖率到 80% 就行，剩下不重要" | 覆盖率是均值，关键路径未覆盖即使 95% 也是裸奔；要看的是「哪 20% 没覆盖」 |
| "这个用例太边角，不值得测" | 边角用例正是用户最容易踩到且最难复现的场景；线上 P0 大都来自被忽略的边角 |
| "测试挂了八成是测试本身的问题，先 skip" | skip 是 flaky 的温床；先 skip 后修在多数项目意味着永远不修。要么修要么删，不留 skip |
| "上线后再补测试也来得及" | 「来得及」=有人专门腾出时间；现实是新需求追着上，测试债只滚不还 |
| "测了行为又测实现是双保险" | 双重测试在重构时双倍踩雷；测试要么测对外契约，要么测内部不变量，不要混淆层次 |
