## Common Rationalizations

> ⚠️ 非穷举清单 — LLM 可能用未列措辞绕过；Master 二次 review 是最后兜底。

| 借口 | 反驳 |
|------|------|
| "这个交互看起来很直观，不需要用研" | 「对设计师直观」≠「对用户直观」；没有数据的直觉是最危险的假设 |
| "设计稿只要视觉好看就行，细节开发自己处理" | 开发阶段的「自行处理」= 交互随意降级；边缘 case 和错误状态必须在设计稿中明确 |
| "移动端适配之后再做，先做桌面端" | 超过 60% 的用户在移动端；后适配意味着重构组件布局，成本是设计期的 3-5 倍 |
| "这个错误提示随便写个英文就行" | 错误提示是用户压力最高的时刻；模糊的错误提示直接损害信任度和转化率 |
| "设计系统先跳过，做完功能再统一" | 没有设计系统的界面是视觉债；每次新页面都在重复造轮子 + 引入不一致 |
| "用户会适应这个交互的" | 用户不会主动学习难用的产品；他们会直接离开或永不再来 |
