# EKET 快速开始示例

本目录包含 EKET 框架使用的完整示例流程。

---

## 示例 1: 功能开发流程

### 场景：实现用户登录功能

#### 1. 人类输入需求

在 `inbox/human_input.md` 中填写：

```markdown
# 我想要

实现用户登录功能，支持用户名密码登录。

## 背景/目的

系统需要用户认证功能，只有登录用户才能访问个人数据。

## 期望结果

- 用户可以输入用户名和密码登录
- 登录成功后跳转到个人主页
- 登录状态保持 7 天

## 验收标准

- [ ] 登录表单有用户名和密码输入框
- [ ] 密码输入显示为星号
- [ ] 登录成功显示欢迎消息
- [ ] 登录失败显示错误提示
- [ ] 7 天内自动保持登录状态
```

#### 2. 智能体分析并创建任务

智能体会在 `tasks/` 目录创建任务文件：

```markdown
# Task: 实现用户登录功能

id: task-001
status: in_progress
priority: normal

## 任务拆解

1. 创建登录表单组件
2. 实现登录 API 调用
3. 添加表单验证
4. 编写单元测试

## 分支

feature/task-001-user-login
```

#### 3. 智能体执行开发

智能体创建分支并开发：
```bash
git checkout -b feature/task-001-user-login
# 开发工作...
git add .
git commit -m "feat(auth): implement user login form"
git push -u origin feature/task-001-user-login
```

#### 4. 智能体提交 Review 请求

在 `outbox/review_requests/review-task-001.md` 创建：

```markdown
# Review 请求

**任务 ID**: task-001
**请求时间**: 2026-03-19 15:00
**状态**: pending_review

---

## 变更内容

- 新增 LoginForm 组件
- 新增 login API 调用
- 添加表单验证逻辑

## 验收标准

- [x] 登录表单有用户名和密码输入框
- [x] 密码输入显示为星号
- [x] 登录成功显示欢迎消息
- [x] 登录失败显示错误提示
- [ ] 7 天内自动保持登录状态 (下一步实现)

## PR

#1 - Implement user login form
```

#### 5. 智能体创建状态报告

在 `inbox/human_feedback/status-task-001.md` 创建：

```markdown
# 任务状态报告

**任务 ID**: task-001
**时间**: 2026-03-19 15:00
**阶段**: 开发完成，等待 Review
**状态**: `pending_confirmation`

---

## 本阶段成果

- 完成登录表单组件
- 完成登录 API 集成
- 通过所有单元测试

## 待确认问题

### 问题 1: 登录状态保持方式

| 选项 | 描述 | 优缺点 |
|------|------|--------|
| A | localStorage | 简单，但安全性较低 |
| B | HttpOnly Cookie | 安全，但需要后端配合 |

**推荐**: 选项 B - 更安全，符合最佳实践

## 下一步计划

1. 根据 Review 反馈修改
2. 实现登录状态保持
3. 完善错误处理
```

#### 6. 人类回复反馈

在同一文件中回复：

```markdown
## 人类反馈

**回复于**: 2026-03-19 16:00

### 确认状态

- [x] 确认阶段成果
- [x] 同意技术方案（选择选项 B）
- [x] 批准进入下一阶段

### 额外指示

请先完成 7 天登录状态保持功能，再进行 Review。
```

#### 7. 智能体继续执行

智能体根据反馈继续实现剩余功能。

#### 8. 任务完成

最终任务状态更新为 `done`。

## 示例 2: Bug 修复流程

### 场景：修复登录页面加载缓慢问题

#### 1. 人类输入

```markdown
# 我想要

修复登录页面加载缓慢的问题。

## 背景/目的

用户反馈登录页面需要 3-5 秒才能加载完成。

## 期望结果

页面加载时间不超过 1 秒。

## 验收标准

- [ ] 首次加载时间 < 1s
- [ ] Lighthouse 性能评分 > 90
```

#### 2. 智能体分析

智能体分析性能瓶颈，可能发现：
- 图片未压缩
- 未使用代码未删除
- 未启用缓存

#### 3. 创建状态报告

```markdown
# 任务状态报告

**任务 ID**: bug-001
**阶段**: 性能分析完成

## 本阶段成果

- 完成性能分析
- 识别 3 个瓶颈

## 待确认问题

### 问题 1: 优化优先级

1. 图片压缩（预计提升 40%）
2. 代码分割（预计提升 30%）
3. 启用缓存（预计提升 20%）

**推荐**: 按上述顺序依次优化

## 下一步计划

依次执行三项优化
```

## 常用命令参考

```bash
# 查看状态
/eket-status

# 创建任务
/eket-task 实现用户注册功能

# 领取任务
/eket-claim task-001

# 请求 Review
/eket-review task-001

# 获取帮助
/eket-help
```

## 文件命名规范

| 文件类型 | 命名规范 | 示例 |
|---------|---------|------|
| 状态报告 | `status-<task-id>-<timestamp>.md` | `status-task-001-20260319.md` |
| Review 请求 | `review-<task-id>.md` | `review-task-001.md` |
| 人类反馈 | `<original-file-name>-feedback.md` | `status-task-001-feedback.md` |
