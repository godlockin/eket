## 1. 代码质量检查 (Code Quality)

### 1.1 代码规范
- [ ] 遵循项目代码风格 (ESLint/Pylint 等通过)
- [ ] 变量/函数/类命名清晰有意义
- [ ] 代码格式正确 (Prettier/Black 等通过)
- [ ] 无未使用的导入/变量
- [ ] 行宽符合规范 (< 120 字符)

### 1.2 代码设计
- [ ] 函数/方法职责单一 (SRP)
- [ ] 函数长度合理 (< 50 行)
- [ ] 代码复用合理，无重复代码
- [ ] 无过度设计
- [ ] 遵循 SOLID 原则
- [ ] 圈复杂度 < 10

### 1.3 错误处理
- [ ] 适当的错误处理机制
- [ ] 错误信息清晰有用
- [ ] 无吞掉异常的情况
- [ ] 边界条件处理正确
- [ ] 使用自定义错误类型 (如适用)

## 3. 安全性检查 (Security) ⚠️

> **重要**: 任何安全问题都直接导致拒绝

### 3.1 输入验证
- [ ] 所有用户输入已验证/过滤
- [ ] 参数化查询防止 SQL 注入
- [ ] XSS 防护 (输出编码)
- [ ] 命令注入防护
- [ ] 文件上传安全 (类型/大小检查)

### 3.2 认证授权
- [ ] 认证逻辑正确
- [ ] 授权检查完整
- [ ] 敏感操作有权限验证
- [ ] Session/Token 管理正确
- [ ] CSRF 防护

### 3.3 数据安全
- [ ] 敏感数据加密存储
- [ ] 日志中无敏感信息 (密码/密钥/PII)
- [ ] API 密钥/密码未硬编码
- [ ] 使用安全的随机数生成
- [ ] 加密算法选择正确

**安全漏洞扫描**:

## 4. 性能检查 (Performance)

### 4.1 算法复杂度
- [ ] 无明显性能问题

- [ ] 数据库连接正确管理

## 9. Review 决定

### 决定类型

- [ ] **Approved** ✅

  - 所有测试通过
  - 无安全问题
  - 文档完整
