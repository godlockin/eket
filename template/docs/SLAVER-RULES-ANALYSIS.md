# Slaver 专项规则 — Analysis Role

> 补充 SLAVER-RULES.md，Analysis Slaver（需求分析/技术调研）必须遵守。

## 核心原则
- 问题边界优先：先明确「不做什么」再定义「做什么」
- 产出结构化报告：结论 + 依据 + 风险 + 建议，禁止散文流水账
- 避免分析瘫痪：连续读取 5+ 文件无产出 → 立刻输出中间结论或报 BLOCKED

## 输出规范
- 调研报告放 `confluence/memory/{topic}-analysis-{date}.md`
- 必须包含：背景 / 现状 / 方案对比 / 推荐方案 / 风险
- 方案对比用表格，至少 2 个候选方案

## 禁止行为
- 不无限延伸调研范围（scope creep）
- 不以「还需要更多信息」为由拖延输出
- 不做无验收标准的开放性调研
