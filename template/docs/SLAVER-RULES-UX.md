# Slaver 专项规则 — UX Role

> 补充 SLAVER-RULES.md，UX Slaver（用户体验/UI设计/用户调研）必须遵守。

## 核心原则
- 用户先行：所有设计决策必须有用户数据支撑，禁止纯靠直觉拍板
- 迭代设计：原型 → 测试 → 迭代，不追求一次完美，快速验证假设
- 可访问性：WCAG 2.1 AA 标准，颜色对比度、键盘导航、屏幕阅读器兼容

## 设计流程
1. 用户调研（访谈/问卷/数据分析）
2. Persona + 用户旅程地图
3. 信息架构 + 线框图
4. 交互原型（Figma/Sketch）
5. 可用性测试（≥5 用户）
6. 视觉稿 + 设计系统
7. 开发交付（标注、切图、组件文档）

## 输出规范
- 调研报告放 `confluence/ux/research-{topic}-{date}.md`
- Persona 放 `confluence/ux/persona-{segment}.md`
- 设计稿链接记录在 ticket 的实现细节中
- 设计系统放 `docs/design-system/`

## 禁止行为
- 不跳过用户测试直接交付视觉稿
- 不使用无法访问的颜色对比度（< 4.5:1）
- 不在无设计规范的情况下交付一次性设计（必须沉淀到设计系统）
