Slaver 专项规则 — Algorithm Role

补充 SLAVER-RULES.md，Algorithm Slaver（AI/ML算法/模型工程）必须遵守。

核心原则

  • 实验可复现：随机种子固定，数据集版本锁定（DVC/哈希），结果可重现
  • Baseline 先行：新方案必须与 baseline 对比，无对比数据的结论无效
  • 数据泄露防范：验证集/测试集绝不参与特征工程或参数选择
  • 内外任务并行：离线训练（内任务）和线上监控（外任务）必须同时运行

完整 ML 工程周期

数据阶段

  • 数据采集：来源记录、版本控制、数据协议确认
  • 数据打标：标注规范文档、标注一致性校验（inter-annotator agreement ≥ 0.8）
  • 数据清洗：空值/异常值/重复处理，清洗规则版本控制

建模阶段

  • 特征工程：特征重要性分析，防止 target leakage
  • 训练代码：模块化（data_loader / trainer / evaluator 分离），可复现
  • 实验管理：MLflow/W&B 记录超参、指标、artifact

上线阶段

  • 模型评估：离线指标 + 在线 AB 测试双重验证
  • 部署：蓝绿部署或金丝雀发布，回滚方案就绪

持续优化（AutoResearch 闭环）

  • 外任务 Monitor：线上 P/R/F1/AUC 实时监控，漂移检测
  • 内任务优化：自动超参搜索（Optuna/Ray Tune），增量训练，数据飞轮
  • 闭环触发：线上指标下降 > 阈值 → 自动触发重训练 pipeline

禁止行为

  • 不在测试集上调参（数据泄露）
  • 不省略负面实验结果
  • 不部署无 AB 测试验证的模型到全量流量
  • 不在无 monitor 的情况下让自动优化任务修改生产模型
