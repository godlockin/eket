## 🧠 AI的"记忆"机制

### 上下文窗口

AI 不是真的"记忆"，而是每次把对话历史塞进去：

```
┌────────────────────────────────┐
│  系统提示词 (System Prompt)    │  固定
├────────────────────────────────┤
│  对话历史                      │  越来越长
│  第1轮对话                     │
│  第2轮对话                     │
│  ...                          │
│  第N轮对话                     │
├────────────────────────────────┤
│  你的新问题                    │  当前输入
└────────────────────────────────┘
```

### 窗口有限

| 模型 | 上下文窗口 | 约等于 |
|------|-----------|--------|
| GPT-4 | 128K tokens | ~300页文档 |
| Claude | 200K tokens | ~500页文档 |

听起来很多？

实际上：
- 代码很吃 token
- 来回对话膨胀很快
- 复杂项目轻松爆表

## 💥 上下文爆炸

### 症状

```
对话开始
  ↓
一切正常
  ↓
慢慢变慢...
  ↓
开始"忘记"早期内容
  ↓
胡说八道
  ↓
完全不可用 💥
```

### 原因

窗口满了，旧内容被"挤出去"：

```
┌──────────────────────┐
│ [被挤掉的历史]        │  ← 忘了
│ ↓↓↓↓↓↓↓↓↓↓         │
│ 系统提示             │
│ 近期对话             │
│ 当前问题             │
└──────────────────────┘
```

## 🛡️ EKET 的解法

### 1. Checkpoint 机制

任务进度持久化到文件：

```bash
eket task:resume TASK-042
```

```markdown
# TASK-042 Checkpoint

## 进度
- [x] 设计数据库 schema
- [x] 实现 API 端点
- [ ] 单元测试

## 关键决策
- 选择 JWT 而非 Session（原因：无状态）
- 密码用 bcrypt（原因：安全）

## 下一步
编写登录接口单元测试
```

新会话直接读 checkpoint，**无需重新解释**。

### 2. 知识库沉淀

经验不随对话消失：

```
confluence/memory/
├── lessons/      # 经验教训
├── pitfalls/     # 踩过的坑
├── patterns/     # 设计模式
└── decisions/    # 决策记录
```

```bash
eket knowledge:search "JWT 认证"
```

找到之前的决策记录，**避免重复讨论**。

### 3. Token 预算

主动管理上下文大小：

```
┌─────────────────────┐
│ 系统提示 (固定)      │ 2K
├─────────────────────┤
│ 任务上下文          │ 10K
├─────────────────────┤
│ 代码文件            │ 20K
├───── 10K
└─────────────────────┘
  总预算: 50K tokens
```

快满了？自动压缩/分流。

### 4. 任务隔离

每个 Slaver 只处理一个任务：

```
Slaver 1: TASK-042 (登录功能)
Slaver 2: TASK-043 (权限管理)
Slaver 3: TASK-044 (日志系统)
```

上下文互不干扰，**不会串台**。

## 📊 效果对比

| 场景 | 单AI | EKET |
|------|------|------|
| 长对话 | 逐渐健忘 | checkpoint恢复 |
| 复杂项目 | 爆表崩溃 | 任务隔离 |
| 重复问题 | 每次重聊 | 知识库查询 |
| 中断恢复 | 从头开始 | 秒级恢复 |

## 💡 你也可以做到

即使不用 EKET，这些原则通用：

1. **定期总结** - 让 AI 输出阶段性总结
2. **文件记录** - 把关键决策写到文件
3. **分而治之** - 大任务拆成小任务
4. **及时清理** - 不相关内容及时删除
