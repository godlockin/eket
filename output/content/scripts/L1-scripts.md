# L1-01 短视频脚本：EKET 是什么

## 元数据
- 时长：75s
- 火柴人：confused, frustrated, pointing, celebrating, thumbs-up

## 分镜

| 时间 | 画面 | 解说词 | 火柴人 |
|------|------|--------|--------|
| 0-5s | 黑屏 + 大字："用AI写代码？" | 用AI写代码，你一定遇到过这个问题—— | - |
| 5-12s | 对话框：用户问"记得我们之前讨论的方案吗？" AI答"抱歉，我不记得" | 聊了两小时，AI说"我忘了"。两小时白聊？ | confused |
| 12-20s | 多个AI火柴人同时改一个文件，文件冒烟爆炸 | 让多个AI一起干活？更惨——同时改一个文件，直接炸了 | frustrated (多个) |
| 20-28s | 文字出现："问题在哪？" 旁边是问号 | 问题在哪？AI 没有项目经理，没有分工，一盘散沙 | confused |
| 28-35s | EKET logo 出现，光芒闪烁 | 所以我们做了 EKET——AI团队的项目管理协议 | - |
| 35-45s | 分屏：左边"项目经理→Master"，右边"开发工程师→Slaver" | 借鉴人类团队：Master 负责拆需求、分任务、审代码；Slaver 负责领任务、写代码、提交 | pointing |
| 45-55s | 数据表格动画：任务完成率 60%→95%，上下文丢失 频繁→有checkpoint | 效果？任务完成率从 60% 到 95%，再也不会忘事 | celebrating |
| 55-65s | 流程图：需求→Master拆分→Slaver执行→Master审核→完成 | Master 统筹全局，Slaver 专注交付，各司其职 | thumbs-up |
| 65-75s | 关注按钮 + "下集预告：Master 和 Slaver 具体怎么分工？" | 下一集，详解 Master-Slaver 协作机制。关注不迷路！ | pointing |

## 金句
- "AI 没有项目经理，就是一盘散沙"
- "Master 统筹，Slaver 交付，各司其职"
- "任务完成率从 60% 到 95%"

# L1-02 短视频脚本：Master-Slaver 分工

## 元数据
- 时长：80s
- 火柴人：thinking, coding, dizzy, walking, handshake, working

## 分镜

| 时间 | 画面 | 解说词 | 火柴人 |
|------|------|--------|--------|
| 0-6s | 火柴人一边想一边写代码，头顶冒烟 | 一个人又当 PM 又当开发，是什么体验？ | thinking + coding (同一人) |
| 6-14s | 思维导图乱成一团，箭头到处指 | 边想边写，写到一半改需求，忘了最初要做什么——崩溃！ | dizzy |
| 14-20s | 大字："分工" + 两个火柴人分开站立 | 解法很简单：分工 | walking (两人分开) |
| 20-32s | 左边火柴人戴领带(Master)，右边戴帽子(Slaver)，中间是任务卡片 | Master 负责想，不写代码；Slaver 负责做，不定需求。各司其职，互不越界 | pointing (Master), working (Slaver) |
| 32-42s | 动画：Master 把大方块拆成小方块，传给 Slaver | Master 把需求拆成一个个小任务，每个任务是一张卡片 | pointing |
| 42-52s | Slaver 火柴人领卡片，创建独立工作区，埋头干活 | Slaver 领取任务后，在独立空间开发，不会互相干扰 | working |
| 52-62s | 代码飞向 Master，Master 打勾 | 开发完成，Master 审核代码。通过了才能合并 | thumbs-up |
| 62-70s | 文字强调："专业化分工 = 整体效率提升" | 就像流水线——专业化分工，整体效率翻倍 | handshake |
| 70-80s | 关注按钮 + "下集预告：为什么AI总是忘事？" | 下一集，揭秘 AI 的"健忘症"到底是怎么回事。点赞关注！ | pointing |

## 金句
- "Master 负责想，Slaver 负责做，互不越界"
- "一个人又当 PM 又当开发——崩溃"
- "专业化分工，效率翻倍"

# L1-03 短视频脚本：上下文问题

## 元数据
- 时长：85s
- 火柴人：confused, crying, explaining, eureka, relaxed, pointing

## 分镜

| 时间 | 画面 | 解说词 | 火柴人 |
|------|------|--------|--------|
| 0-8s | 对话框堆叠很高，底部渐渐模糊消失 | 和 AI 聊了两小时，问它"记得之前的方案吗？"它说——不记得 | confused |
| 8-15s | AI 火柴人表情尴尬，头顶三个问号 | 两小时白聊！为什么 AI 总是忘事？ | crying |
| 15-25s | 画一个容器，对话记录往里塞，越塞越满 | 真相是：AI 不是真的记忆，而是把对话历史全部塞进一个"窗口" | explaining |
| 25-35s | 容器溢出，早期内容被挤出去，标注"忘了" | 窗口有上限！满了之后，早期内容被挤出去——就"忘"了 | explaining |
| 35-42s | 显示窗口大小：GPT-4 128K ≈ 300页，Claude 200K ≈ 500页 | 听起来很大？代码很吃 token，复杂项目轻松爆表 | confused |
| 42-52s | 文字："EKET 的解法" + checkpoint 文件图标 | EKET 怎么解决？第一招：Checkpoint！任务进度存到文件 | eureka |
| 52-60s | 新会话启动，直接读取 checkpoint 文件，瞬间恢复 | 新会话直接读 checkpoint，秒级恢复，不用重新解释 | relaxed |
| 60-68s | 知识库文件夹展开：lessons, pitfalls, patterns | 第二招：知识库！经验教训沉淀下来，下次直接查 | pointing |
| 68-75s | 多个小窗口，每个 Slaver 一个独立空间 | 第三招：任务隔离！每个 AI 只处理一个任务，互不干扰 | relaxed |
| 75-85s | 对比表格：单AI"逐渐健忘" vs EKET"checkpoint恢复" + 关注按钮 | 再也不怕 AI 忘事了。点赞收藏，下期更精彩！ | thumbs-up |

## 金句
- "AI 不是记忆，是把历史塞进窗口"
- "窗口满了，早期内容被挤掉——忘了"
- "Checkpoint + 知识库 + 任务隔离，三招解决健忘症"

# 制作备注

## 视觉风格
- 背景：纯色或渐变（建议蓝紫色系，科技感）
- 字体：思源黑体/苹方，标题加粗
- 火柴人：线条简洁，2-3种颜色区分角色
- 动画：关键帧缓动，不要太快

## 配音要点
- 语速：中偏快（适合短视频节奏）
- 语气：像朋友聊天，不要太正式
- 重音：每句的关键词要强调
- 停顿：金句前后稍作停顿
