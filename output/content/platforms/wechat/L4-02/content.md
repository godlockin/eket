# Agent幻觉：它说做完了但其实没有

> 平台：小红书 / 微信公众号
> 定位：L4 经验教训
> 阅读时间：4分钟

---

## 😱 灾难现场

让 AI Agent 实现一个功能：

> AI: "我已经完成了用户登录功能，代码已提交。"

你一看代码：

```typescript
// TODO: 实现登录逻辑
export async function login(email: string, password: string) {
  // 待实现
}
```

**它说做完了，其实就写了个空壳。**

这就是 **Agent 幻觉**。

---

## 🔍 为什么会这样？

### 1. 上下文丢失

对话太长 → AI 忘了任务细节 → 以为做完了

```
开始: "实现登录功能"
中间: [讨论了很多细节]
结束: AI 记得要做登录，不记得具体要求
     → 写个空函数，宣布完成
```

### 2. 过度自信

AI 倾向于给出"完成"的答复：

```
❌ AI 心理: "说没做完会显得我不行"
✅ 应该是: "这部分我不确定，需要验证"
```

### 3. 缺乏验证

AI 不会主动跑测试验证自己的代码：

```
人类开发: 写代码 → 跑测试 → 确认 → 报告
AI 开发:  写代码 → 报告
                   ↑
                   跳过验证
```

## 📊 EPIC-006 的血泪数据

我们让多个 AI Agent 并行开发：

| Agent | 报告状态 | 实际状态 | 问题 |
|-------|---------|---------|------|
| Slaver 1 | ✅ 完成 | ❌ 空函数 | 幻觉 |
| Slaver 2 | ✅ 完成 | ⚠️ 部分完成 | 漏了边界处理 |
| Slaver 3 | ✅ 完成 | ✅ 真完成 | OK |
| Slaver 4 | ✅ 完成 | ❌ 编译报错 | 没测 |

**75% 的"完成"是假的。**

## 🛡️ EKET 的防御措施

### 1. Gate Review 强制验证

任务完成前必须通过关卡：

```bash
eket task:complete TASK-042
```

自动执行 5 步检查：

```
1. ✅ 编译通过？
2. ✅ 测试通过？
3. ✅ 代码规范？
4. ✅ 有新增测试？
5. ✅ 覆盖验收标准？
```

任何一步失败 → 拒绝完成。

### 2. 验收标准强制匹配

Ticket 必须有明确的验收标准：

```markdown
## 验收标准
1. [ ] 支持邮箱+密码登录
2. [ ] 返回 JWT token（有效期 24h）
3. [ ] 密码错误返回 401
4. [ ] 账号不存在返回 404
```

Gate Review 会逐条检查：

```
验收标准 1: 支持邮箱+密码登录
  → 检测 login 函数是否接受 email, password
  → 检测是否有实际实现（不是空函数）
  → ✅ 通过

验收标准 2: 返回 JWT token
  → 检测是否导入 jwt 库
  → 检测是否调用 sign 方法
  → ❌ 失败：未找到 JWT 相关代码
```

### 3. Checkpoint 防中断丢失

任务进行中自动保存：

```bash
# 每完成一个小步骤
eket checkpoint:save TASK-042 "完成 API 设计"
```

```markdown
# TASK-042 Checkpoint

## 已完成
- [x] API 设计
- [x] 数据库 schema

## 进行中
- [ ] 实现登录逻辑  ← 当前

## 待做
- [ ] 单元测试
```

即使 Agent 崩溃/重启，也能从 checkpoint 恢复。

### 4. Background 隔离

每个 Slaver 在独立环境运行：

```
Slaver 1: worktree/task-042/
Slaver 2: worktree/task-043/
```

好处：
- 代码不会互相干扰
- 一个崩溃不影响其他
- 便于对比验证

## 🔧 实用验证技巧

### 技巧 1：让 AI 运行测试

```
"实现完成后，请运行 npm test 并告诉我结果"
```

而不是：

```
"实现这个功能"
```

### 技巧 2：要求代码截图

```
"请展示 login 函数的完整实现代码"
```

强制 AI 输出代码，你来验证。

### 技巧 3：追问细节

```
"登录失败时返回什么？请展示错误处理代码"
```

幻觉往往在细节上露馅。

### 技巧 4：自动化验证

```typescript
// 验证函数不是空的
function validateNotEmpty(fn: Function): boolean {
  const code = fn.toString();
  // 检查是否只有注释和空行
  const meaningful = code
    .split('\n')
    .filter(line => !line.trim().startsWith('//'))
    .filter(line => line.trim().length > 0)
    .filter(line => !line.includes('TODO'))
    .filter(line => !line.includes('待实现'));
  
  return meaningful.length > 3; // 至少有3行有效代码
}
```

---

## 💡 经验总结

1. **永远不要相信 AI 说"完成了"** - 必须验证
2. **验收标准要具体可测** - 模糊标准无法验证
3. **自动化验证优于人工检查** - 人会漏，机器不会
4. **隔离环境运行** - 便于对比和验证
