## 🚀 v2.3.0 迭代任务

### Sprint 1: 测试修复完善 (3-4 天)

#### Task 1.1: 模块路径问题修复
**负责**: Slaver B (QA Specialist)
**优先级**: P0
**预估**: 2 小时

**问题**:
```
Cannot find module '../../../src/integration/openclaw-adapter'
Cannot find module 'core/master-context'
```

**方案**:
1. 检查文件是否存在，不存在则创建 stub
2. 修复导入路径（绝对路径 vs 相对路径）
3. 更新 jest.config 的 moduleNameMapper

#### Task 1.2: Redis 连接超时修复
**负责**: Slaver D (DevOps Engineer)
**优先级**: P0
**预估**: 3 小时

**问题**:
```
collaboration.test.ts - Exceeded timeout of 5000 ms
```

**方案**:
1. 使用 Redis Mock 替代真实连接（已实现 `tests/helpers/redis-mock.ts`）
2. 增加测试超时到 10000ms (临时方案)
3. 优化 CommunicationProtocol 的连接逻辑

#### Task 1.3: 环境依赖测试改造
**负责**: Slaver D (DevOps Engineer)
**优先级**: P1
**预估**: 4 小时

**目标**: 所有测试都能在无外部依赖下运行

**方案**:
1. 审计所有失败测试的依赖
2. 使用 Mock/Stub 替代外部依赖
3. 更新 `docker-compose.test.yml` 用于集成测试

#### Task 1.4: 测试用例批量修复
**负责**: Slaver B (QA Specialist)
**优先级**: P0
**预估**: 1 天

**目标**: 测试通过率 47% → 80%+

**策略**:
- 按模块分组修复（core, api, commands, integration）
- 优先修复高价值测试（集成测试、核心模块）
- 使用已有的 Mock/Stub 工具

## 🚨 风险与应对

### 风险 1: 测试修复比预期复杂
**概率**: 中
**影响**: 高
**应对**:
- 优先修复高价值测试（核心模块）
- 降级部分低优先级集成测试
- 允许 v2.3.0 达成 90% 而非 100%

### 风险 2: 性能未达预期
**概率**: 低
**影响**: 中
**应对**:
- 4 项优化已实施，风险较低
- 如未达标，可追加优化任务
- 性能调优可延续到 v2.4.0

### 风险 3: SQLite 迁移引入回归
**概率**: 中
**影响**: 高
**应对**:
- 保持向后兼容，旧接口不删除
- 充分测试迁移后的功能
- 灰度迁移策略（部分模块先迁移）
