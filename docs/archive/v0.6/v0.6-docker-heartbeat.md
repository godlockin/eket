# EKET v0.6.0 Docker 集成和 Slaver 心跳监控

**版本**: v0.6.0
**更新时间**: 2026-03-24

---

## 概述

v0.6.0 版本引入了 Docker 容器化支持和 Slaver 心跳监控机制，主要功能包括：

1. **Docker 环境检测** - 自动检测 Docker 安装和运行状态
2. **Docker SQLite 容器** - 用于 confluence 文件索引和 jira 管理
3. **Docker Redis 容器** - 用于 Slaver 心跳存储
4. **Slaver 进程注册** - 进程信息注册到文件和数据库
5. **心跳监控** - 定期心跳上传，超时自动杀进程

---

## 新增脚本

### 1. `scripts/check-docker.sh` - Docker 环境检测

**用途**: 检测 Docker 是否安装并运行

**用法**:
```bash
# 完整检查
./scripts/check-docker.sh

# 静默检查（仅返回状态码）
./scripts/check-docker.sh --silent

# 显示详细信息
./scripts/check-docker.sh --verbose
```

**返回值**:
- `0` - Docker 可用
- `1` - Docker 未安装
- `2` - Docker 未运行
- `3` - Docker Compose 不可用

---

### 2. `scripts/docker-sqlite.sh` - SQLite 容器管理

**用途**: 启动/停止/管理 SQLite 容器，用于 confluence 文件索引和 jira 管理

**用法**:
```bash
# 启动容器
./scripts/docker-sqlite.sh start

# 停止容器
./scripts/docker-sqlite.sh stop

# 重启容器
./scripts/docker-sqlite.sh restart

# 查看状态
./scripts/docker-sqlite.sh status

# 查看日志
./scripts/docker-sqlite.sh logs

# 删除容器（保留数据）
./scripts/docker-sqlite.sh remove

# 导出配置
./scripts/docker-sqlite.sh export
```

**配置输出**: `.eket/config/docker-sqlite.yml`

---

### 3. `scripts/docker-redis.sh` - Redis 容器管理

**用途**: 启动/停止/管理 Redis 容器，用于 Slaver 心跳存储

**用法**:
```bash
# 启动容器
./scripts/docker-redis.sh start

# 停止容器
./scripts/docker-redis.sh stop

# 重启容器
./scripts/docker-redis.sh restart

# 查看状态
./scripts/docker-redis.sh status

# 查看日志
./scripts/docker-redis.sh logs

# 清空数据
./scripts/docker-redis.sh flush

# 导出配置
./scripts/docker-redis.sh export
```

**默认配置**:
- 端口：`6380`
- 密码：`eket_redis_2026`
- 容器名：`eket-redis`

**配置输出**: `.eket/config/docker-redis.yml`

---

### 4. `scripts/slaver-heartbeat.sh` - Slaver 进程注册和心跳

**用途**: Slaver 进程注册到文件、心跳上传、支持 Redis 和 SQLite 备份

**用法**:
```bash
# 注册进程
./scripts/slaver-heartbeat.sh register <slaver-name> <task-id> [port]

# 发送心跳
./scripts/slaver-heartbeat.sh heartbeat <slaver-name>

# 查询状态
./scripts/slaver-heartbeat.sh status <slaver-name>

# 列出所有 Slaver
./scripts/slaver-heartbeat.sh list

# 清理超时 Slaver
./scripts/slaver-heartbeat.sh cleanup

# 启动心跳守护进程
./scripts/slaver-heartbeat.sh start-daemon <slaver-name>

# 停止心跳守护进程
./scripts/slaver-heartbeat.sh stop-daemon <slaver-name>
```

**环境变量**:
- `HEARTBEAT_INTERVAL` - 心跳间隔（默认：30 秒）
- `HEARTBEAT_TIMEOUT` - 超时阈值（默认：300 秒）

**注册信息**:
- 进程 PID
- Slaver 名称
- 任务 ID
- 主机名
- 端口
- 启动时间
- 最后心跳时间
- 心跳计数

**存储位置**:
1. 文件：`.eket/state/slavers/<slaver-name>.yml`
2. Redis: `slaver:<slaver-name>` hash
3. SQLite: `slaver_processes` 表

---

### 5. `scripts/heartbeat-monitor.sh` - 心跳监控（v0.6.0 升级）

**用途**: 监控 Slaver 心跳，检测超时并触发告警、重置和杀进程

**升级内容**:
- 增加进程终止功能（SIGTERM → SIGKILL）
- 从 Redis 删除超时记录
- 从 SQLite 删除超时记录
- 支持新版 slavers 目录

**用法**:
```bash
# 单次检查
./scripts/heartbeat-monitor.sh

# 守护模式
./scripts/heartbeat-monitor.sh --daemon

# 手动检查
./scripts/heartbeat-monitor.sh --check
```

**超时处理流程**:
1. 检测心跳超时（>300 秒）
2. 发送 SIGTERM 信号到进程
3. 等待 10 秒，如未退出则发送 SIGKILL
4. 从 Redis 删除记录
5. 从 SQLite 删除记录
6. 清理状态文件
7. 重置任务状态为 ready
8. 创建告警文件
9. 通知 Master 和人类

## 集成到启动流程

### `template/.claude/commands/eket-init.sh`

系统初始化时自动检测和启动 Docker 容器：

```bash
# 步骤 3.5: Docker 环境检测
if ./scripts/check-docker.sh --silent; then
    # 启动 SQLite 容器
    ./scripts/docker-sqlite.sh start

    # 启动 Redis 容器
    ./scripts/docker-redis.sh start
fi
```

### `template/.claude/commands/eket-start.sh`

Slaver 模式启动时自动注册进程和启动心跳：

```bash
# v0.6.0: Docker 环境检测
if ./scripts/check-docker.sh --silent; then
    # 启动容器（如果未运行）
    ./scripts/docker-sqlite.sh start
    ./scripts/docker-redis.sh start
fi

# v0.6.0: Slaver 进程注册
./scripts/slaver-heartbeat.sh register "$SLAVER_NAME" "$CURRENT_TASK"

# 启动心跳守护进程
./scripts/slaver-heartbeat.sh start-daemon "$SLAVER_NAME"
```

## 配置说明

### Docker SQLite 配置

文件：`.eket/config/docker-sqlite.yml`

```yaml
sqlite:
  container_name: eket-sqlite
  port: 8080
  data_dir: /path/to/project/.eket/data/sqlite
  database: /path/to/project/.eket/data/sqlite/eket.db
  status: running
```

### Docker Redis 配置

文件：`.eket/config/docker-redis.yml`

```yaml
redis:
  container_name: eket-redis
  image: redis:7-alpine
  port: 6380
  password: eket_redis_2026
  data_dir: /path/to/project/.eket/data/redis
  config_file: /path/to/project/.eket/data/redis/redis.conf
  status: running

connection:
  host: localhost
  port: 6380
  password: eket_redis_2026
  url: redis://:eket_redis_2026@localhost:6380
```

### Slaver 状态文件

文件：`.eket/state/slavers/<slaver-name>.yml`

```yaml
slaver_name: slaver_hostname_12345
pid: 12345
task_id: FEAT-001
host: myhost
port: null
status: active
started_at: 2026-03-24T09:00:00+08:00
last_heartbeat: 2026-03-24T09:05:00+08:00
heartbeat_count: 10

resources:
  cpu_usage: 0
  memory_usage: 0

metadata:
  hostname: myhost
  username: user
  working_dir: /path/to/project
```

## 数据库表结构

### SQLite - slaver_processes 表

```sql
CREATE TABLE slaver_processes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    slaver_name TEXT UNIQUE NOT NULL,
    pid INTEGER NOT NULL,
    task_id TEXT,
    host TEXT NOT NULL,
    port INTEGER,
    started_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_heartbeat TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status TEXT DEFAULT 'active',
    metadata TEXT
)
```

### Redis - slaver:<name> Hash

```
HSET slaver:slaver_name pid 12345
HSET slaver:slaver_name task_id FEAT-001
HSET slaver:slaver_name host myhost
HSET slaver:slaver_name port 8080
HSET slaver:slaver_name status active
HSET slaver:slaver_name started_at 2026-03-24T09:00:00+08:00
HSET slaver:slaver_name last_heartbeat 2026-03-24T09:05:00+08:00
HSET slaver:slaver_name heartbeat_count 10
EXPIRE slaver:slaver_name 300  # 超时时间
```

## 故障排查

### Slaver 心跳超时

1. 检查 Slaver 进程是否仍在运行：
   ```bash
   ps aux | grep <pid>
   ```

2. 查看 Slaver 日志：
   ```bash
   cat logs/slaver-heartbeat.log
   ```

3. 查看心跳监控日志：
   ```bash
   cat logs/heartbeat-monitor.log
   ```

### Docker 容器无法启动

1. 检查 Docker 状态：
   ```bash
   ./scripts/check-docker.sh --verbose
   ```

2. 查看容器日志：
   ```bash
   docker logs eket-redis
   docker logs eket-sqlite
   ```

3. 检查端口占用：
   ```bash
   lsof -i :6380
   lsof -i :8080
   ```

### Redis 连接失败

1. 测试连接：
   ```bash
   redis-cli -p 6380 -a eket_redis_2026 ping
   ```

2. 检查容器状态：
   ```bash
   docker ps --filter "name=eket-redis"
   ```

3. 查看 Redis 配置：
   ```bash
   cat .eket/config/docker-redis.yml
   ```

## 安全注意事项

1. **密码修改**: 生产环境请修改默认 Redis 密码
2. **端口暴露**: 默认配置仅允许本地访问，如需远程访问请修改防火墙规则
3. **数据持久化**: Redis 和 SQLite 数据卷挂载到本地，确保定期备份
4. **容器隔离**: 建议使用 Docker 网络隔离生产环境
