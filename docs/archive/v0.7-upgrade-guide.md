## 概述

EKET v0.7 引入了 Node.js 混合架构，在保持原有 Shell 脚本功能的基础上，增加了高级功能支持：

### 新增功能

| 功能 | 实现方式 | 说明 |
|------|---------|------|
| Redis 客户端 | Node.js | Slaver 心跳监控、消息队列 |
| SQLite 客户端 | Node.js | Retrospective 数据持久化 |
| CLI 框架 | Node.js + Commander | 统一的命令行界面 |
| 混合适配器 | Shell | 自动路由和降级 |

## 升级步骤

### 步骤 1: 检查先决条件

```bash
# 检查 Node.js 是否安装
node -v

# 需要 >= 18.0.0
# 如果未安装，请先安装 Node.js
```

**安装 Node.js**:

```bash
# macOS
brew install node@20

# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Windows
# 从 https://nodejs.org 下载安装包
```

### 步骤 2: 启用高级功能

```bash
# 在项目根目录执行
./scripts/enable-advanced.sh
```

该脚本会：
1. 检查 Node.js 环境
2. 安装 NPM 依赖
3. 构建 TypeScript 项目
4. 验证安装

### 步骤 3: 验证安装

```bash
# 检查 Node.js 模块
node node/dist/index.js check

# 系统诊断
./lib/adapters/hybrid-adapter.sh doctor
```

### 步骤 4: 启动 Redis 和 SQLite（可选）

```bash
# 启动 Redis 容器
./scripts/docker-redis.sh start

# 初始化 SQLite 数据库
./scripts/docker-sqlite.sh init
```

## 使用示例

### Redis 相关

```bash
# 检查 Redis 连接
./lib/adapters/hybrid-adapter.sh redis:check

# 列出活跃 Slaver
./lib/adapters/hybrid-adapter.sh redis:list-slavers
```

### SQLite 相关

```bash
# 检查 SQLite 数据库
./lib/adapters/hybrid-adapter.sh sqlite:check

# 列出所有 Retrospective
./lib/adapters/hybrid-adapter.sh sqlite:list-retros

# 搜索 Retrospective
./lib/adapters/hybrid-adapter.sh sqlite:search "测试"

# 生成统计报告
./lib/adapters/hybrid-adapter.sh sqlite:report
```

### 直接使用 Node.js

```bash
# 也可以直接调用 Node.js CLI
node node/dist/index.js redis:check
node node/dist/index.js sqlite:report
```

## 故障排除

### 问题 1: 安装失败

**症状**: `npm install` 失败

**解决方案**:
```bash
# 清理并重新安装
./scripts/enable-advanced.sh --clean

# 或手动操作
cd node/
rm -rf node_modules package-lock.json
npm install
npm run build
```

### 问题 2: Redis 连接失败

**症状**: `redis:check` 返回连接错误

**解决方案**:
```bash
# 检查 Redis 容器状态
docker ps | grep redis

# 启动 Redis 容器
./scripts/docker-redis.sh start

# 检查 Redis 配置
echo $EKET_REDIS_HOST
echo $EKET_REDIS_PORT
```

### 问题 3: TypeScript 编译错误

**症状**: `npm run build` 失败

**解决方案**:
```bash
# 检查 TypeScript 版本
node -v
npm -v

# 确保使用 Node.js >= 18.0.0
# 如果版本过旧，请升级 Node.js
```

### 问题 4: 命令找不到

**症状**: `command not found: hybrid-adapter.sh`

**解决方案**:
```bash
# 确保脚本有执行权限
chmod +x lib/adapters/hybrid-adapter.sh
chmod +x scripts/enable-advanced.sh
```

## 环境变量

以下环境变量可用于配置：

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `EKET_REDIS_HOST` | Redis 主机 | `localhost` |
| `EKET_REDIS_PORT` | Redis 端口 | `6379` |
| `EKET_REDIS_PASSWORD` | Redis 密码 | 无 |
| `EKET_SQLITE_PATH` | SQLite 数据库路径 | `~/.eket/data/sqlite/eket.db` |

## 回滚方案

如果需要回滚到纯 Shell 版本：

```bash
# 1. 删除 Node.js 构建产物
rm -rf node/dist/

# 2. 保留 node_modules（可选）
# mv node/node_modules node/node_modules.bak

# 3. 所有 Shell 脚本仍然可用
./scripts/start.sh
./scripts/docker-redis.sh
./scripts/docker-sqlite.sh
```
