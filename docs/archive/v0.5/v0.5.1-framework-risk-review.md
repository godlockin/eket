## 二、Critical 风险 (🔴)

### C-01: YAML 配置语法错误

**风险描述**: `advanced.yml` 第 79-81 行使用中文字符作为 YAML key，违反 YAML 语法规范。

**影响**:
- 配置加载失败
- 脚本启动报错
- 整个错误处理规范无法生效

**当前代码问题**:
```yaml
# template/.eket/config/advanced.yml 第 77-81 行
error_handling:
  enabled: true
  统一 error 码：true      # ❌ 非法 YAML key
  统一日志格式：true      # ❌ 非法 YAML key
  统一退出码：true        # ❌ 非法 YAML key
```

**修复建议**:
```yaml
error_handling:
  enabled: true
  unified_error_codes: true
  unified_log_format: true
  unified_exit_codes: true
```

**优先级**: P0 - 立即修复

### C-03: 脚本路径硬编码

**风险描述**: 命令脚本中使用相对路径 `../../scripts/`，目录结构变化时失效。

**影响**:
- 项目初始化后脚本无法找到
- 不同部署环境路径不一致
- 维护困难

**当前代码问题**:
```bash
# eket-claim.sh 第 148 行
if [ -x "../../scripts/task-time-tracker.sh" ]; then
    ../../scripts/task-time-tracker.sh start "$TASK_ID" ...

# eket-claim.sh 第 165 行
if [ -x "../../scripts/slaver-permissions.sh" ]; then

# eket-claim.sh 第 182 行
if [ -x "../../scripts/checkpoint-sprint-retro.sh" ]; then
```

**修复建议**:
```bash
# 使用动态路径
SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
SCRIPTS_DIR="$PROJECT_ROOT/scripts"

if [ -x "$SCRIPTS_DIR/task-time-tracker.sh" ]; then
    "$SCRIPTS_DIR/task-time-tracker.sh" start "$TASK_ID" ...
```

**优先级**: P0 - 本周修复

### C-04: Merge 流程绕过验证

**风险描述**: `eket-review-merge.sh` 直接执行 Git 合并，未调用 `merge-strategy.sh` 和 `test-gate-system.sh`。

**影响**:
- 未完成功能可被合并
- 测试未通过可被合并
- Review 验证形同虚设

**当前代码问题**:
```bash
# eket-review-merge.sh 第 277-307 行
# 直接合并，无验证
git checkout testing
git merge "$BRANCH" -m "feat: merge $TASK_ID"

git checkout main
git merge testing -m "feat: merge $TASK_ID to main"
```

**应该调用**:
- `scripts/merge-strategy.sh` - 检查完成率、决定是否部分合并
- `scripts/test-gate-system.sh` - 执行 9 大门禁测试
- `scripts/checkpoint-validator.sh` - 验证 Checkpoint

**修复建议**:
```bash
# 合并前验证
./scripts/test-gate-system.sh all "$TASK_ID" || {
    echo "❌ 测试门禁未通过，阻止合并"
    return 1
}

./scripts/merge-strategy.sh "$TASK_ID" --auto || {
    echo "❌ 合并策略检查失败"
    return 1
}
```

**优先级**: P0 - 立即修复

### C-05: 监控脚本无守护进程管理

**风险描述**: `heartbeatmonitor.sh` 和 `memory-review-agent.sh` 使用简单 `&` 后台运行，无进程管理。

**影响**:
- 进程挂掉后无法自动重启
- 无 PID 文件管理
- 无法优雅关闭

**当前代码问题**:
```bash
# eket-start.sh
../../scripts/heartbeatmonitor.sh --daemon &>/dev/null &
../../scripts/memory-review-agent.sh --daemon &>/dev/null &
```

**问题**:
- 无 PID 文件追踪
- 无重启逻辑
- 实例重启后不会自动恢复

**修复建议**:
```bash
# 使用进程管理
start_daemon() {
    local script="$1"
    local name="$2"
    local pid_file=".eket/state/${name}.pid"

    if [ -f "$pid_file" ]; then
        local pid=$(cat "$pid_file")
        if kill -0 "$pid" 2>/dev/null; then
            echo "✓ $name 已在运行 (PID: $pid)"
            return 0
        fi
    fi

    # 启动并保存 PID
    nohup "$script" --daemon > "logs/${name}.log" 2>&1 &
    echo $! > "$pid_file"
    echo "✓ $name 已启动 (PID: $!)"
}

start_daemon "./scripts/heartbeatmonitor.sh" "heartbeat-monitor"
start_daemon "./scripts/memory-review-agent.sh" "memory-review-agent"
```

**优先级**: P0 - 本周修复

### H-02: 动态覆盖率阈值实现不完整

**当前代码**:
```bash
# test-gate-system.sh 第 90-114 行
get_min_coverage() {
    local ticket_id="$1"
    local ticket_file="$JIRA_DIR"/*/".${ticket_id}.md"

    if [ -f "$ticket_file" ]; then
        local priority=$(grep "^priority:" "$ticket_file" ... )
        local type=$(grep "^type:" "$ticket_file" ... )

        case "$priority" in
            P0|urgent) echo "95" ;;
            P1|high) echo "90" ;;
            P2|normal) echo "80" ;;
            *)
                case "$type" in  # ⚠️ 仅按类型回退
                    feature) echo "80" ;;
                    bugfix) echo "85" ;;
                    hotfix) echo "70" ;;
                    *) echo "80" ;;
                esac
                ;;
        esac
    else
        echo "80"
    fi
}
```

**配置要求**:
```yaml
testing:
  unit_test:
    coverage:
      by_priority:
        P0: 95
        P1: 90
        P2: 80
      by_type:
        feature: 80
        bugfix: 85
        hotfix: 70
      by_module:  # ❌ 未实现
        critical: 95
        important: 90
        normal: 80
```

**修复建议**: 添加模块重要性检测逻辑（需要分析代码路径）。

**优先级**: P1 - 本周修复
