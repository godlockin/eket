## 初始化后首次启动流程

当你运行 `./scripts/init-project.sh` 初始化项目后，打开 Claude Code 并执行 `/eket-init`，会发生以下事情：

### 步骤 1: 项目结构检查

```
✓ .eket
✓ .eket/state
✓ inbox
✓ inbox/human_feedback
✓ outbox
✓ outbox/review_requests
✓ tasks
✓ CLAUDE.md
✓ .eket/config.yml
✓ inbox/human_input.md
```

智能体验证所有必需的文件和目录是否存在。

### 步骤 2: 健康检查

运行 `.eket/health_check.sh`，检查：
- 核心文件完整性
- Claude Code 配置
- 目录结构
- 脚本执行权限
- Git 状态

### 步骤 3: 显示快速开始指南

智能体显示使用指南，告诉你下一步该做什么。

### 步骤 4: 显示当前输入状态

显示 `inbox/human_input.md` 的内容，提示你编辑此文件来描述需求。

## 各阶段详细行为

### 阶段 1: 需求分析

**负责智能体**: 需求分析师 (Requirement Analyst)

**触发条件**:
- `inbox/human_input.md` 被创建或修改
- 用户发送消息触发分析

**智能体行为**:
1. 读取 `inbox/human_input.md` 内容
2. 分析用户需求
3. 识别功能点、非功能需求
4. 拆解为可执行的 Jira tickets
5. 定义验收标准

**输出物**:
- `tasks/task-XXX.md` - 任务定义文件
- `inbox/human_feedback/analysis-<timestamp>.md` - 需求分析状态报告

**状态报告示例**:
```markdown
# 任务状态报告

**任务 ID**: N/A (需求分析阶段)
**时间**: 2026-03-20 10:30
**阶段**: 需求分析完成
**状态**: `pending_confirmation`

---

## 本阶段成果

- 分析了用户输入的需求
- 拆解为 3 个独立任务
- 定义了每个任务的验收标准

## 待确认问题

1. 任务优先级是否正确？
   - 任务 1: 用户登录 (high)
   - 任务 2: 用户注册 (normal)
   - 任务 3: 密码重置 (low)
   - **推荐**: 按上述优先级执行

2. 是否需要先设计数据库 Schema？
   - 选项 A: 先设计 Schema，再开发
   - 选项 B: 边开发边设计
   - **推荐**: 选项 A - 更符合最佳实践

## 下一步计划

1. 等待人类确认任务优先级
2. 创建数据库设计任务
3. 唤醒执行智能体开始开发
```

---

### 阶段 2: 任务领取

**负责智能体**: 执行智能体 (Executor)

**触发条件**:
- 任务状态变为 `ready`
- 执行智能体轮询到可用任务

**智能体行为**:
1. 扫描 `tasks/` 目录
2. 筛选状态为 `ready` 或 `backlog` 的任务
3. 检查自身能力是否匹配任务需求
4. 更新任务状态为 `in_progress`
5. 创建任务领取消息

**输出物**:
- 更新 `tasks/task-XXX.md` 状态
- `shared/message_queue/outbox/task-claimed-XXX.json` - 领取消息

---

### 阶段 3: 开发实现

**负责智能体**: 前端开发/后端开发/设计师/测试员

**触发条件**:
- 任务状态为 `in_progress`
- 智能体已领取任务

**智能体行为**:
1. 创建 Git 分支：`git checkout -b feature/task-XXX-description`
2. 编写代码/设计/测试
3. 提交代码：`git add . && git commit -m "feat: description"`
4. 推送到远程：`git push -u origin feature/task-XXX-description`

**输出物**:
- 代码文件
- 测试文件
- Git 提交记录

---

### 阶段 4: Review 请求

**负责智能体**: 执行智能体

**触发条件**:
- 开发完成
- 代码已提交

**智能体行为**:
1. 创建 PR（通过 Git 命令或 API）
2. 指定协调智能体为 Reviewer
3. 发送 Review 请求消息
4. 创建状态报告等待人类确认

**输出物**:
- `outbox/review_requests/review-task-XXX.md` - Review 请求
- `inbox/human_feedback/dev-complete-<timestamp>.md` - 开发完成状态报告
- GitHub/GitLab PR

**Review 请求示例**:
```markdown
# Review 请求

**任务 ID**: task-001
**请求时间**: 2026-03-20 15:00
**状态**: pending_review

---

## 变更内容

- 新增用户登录表单组件
- 新增登录 API 接口
- 新增表单验证逻辑

## 验收标准

- [x] 登录表单有用户名和密码输入框
- [x] 密码输入显示为星号
- [x] 登录成功显示欢迎消息
- [x] 登录失败显示错误提示
- [ ] 7 天内自动保持登录状态 (下一步实现)

## PR

#1 - Implement user login form

## 测试

- [x] 单元测试通过
- [x] 手动测试完成
```

---

### 阶段 5: Review 执行

**负责智能体**: 技术经理 (Tech Manager)

**触发条件**:
- 收到 Review 请求消息
- PR 被创建

**智能体行为**:
1. 读取 PR 内容
2. 检查代码质量
3. 验证验收标准
4. 决定：通过/驳回

**输出物**:
- PR 评论（通过/修改意见）
- 更新任务状态

---

### 阶段 6: 任务完成

**负责智能体**: 项目经理 (Project Manager)

**触发条件**:
- PR 合并完成
- 所有验收标准满足

**智能体行为**:
1. 更新 Jira 任务状态为 `done`
2. 生成任务总结
3. 归档任务相关文件
4. 通知相关方

**输出物**:
- 更新 `tasks/task-XXX.md` 状态为 `done`
- 任务总结报告

## 命令参考

| 命令 | 功能 | 智能体行为 |
|------|------|----------|
| `/eket-init` | 初始化向导 | 检查项目结构、运行健康检查、显示指南 |
| `/eket-status` | 查看状态 | 显示任务列表、输入输出状态 |
| `/eket-task [desc]` | 创建/查看任务 | 创建新任务或显示任务列表 |
| `/eket-review [id]` | 请求 Review | 创建 Review 请求 |
| `/eket-claim [id]` | 领取任务 | 领取指定任务 |
| `/eket-help` | 帮助 | 显示帮助信息 |

## 示例：完整任务流转

### 用户操作

1. **初始化项目**:
   ```bash
   /path/to/eket/scripts/init-project.sh my-project /path/to/project
   cd /path/to/project
   claude
   /eket-init
   ```

2. **编辑 `inbox/human_input.md`**:
   ```markdown
   # 我想要

   实现用户登录功能，支持用户名密码登录。

   ## 背景/目的

   系统需要用户认证功能。

   ## 期望结果

   - 用户可以输入用户名和密码登录
   - 登录成功后跳转到个人主页

   ## 验收标准

   - [ ] 登录表单有用户名和密码输入框
   - [ ] 密码输入显示为星号
   - [ ] 登录成功显示欢迎消息
   ```

3. **发送消息给 Claude**:
   > "我已经写好了需求，请开始分析"

### 智能体响应

4. **需求分析师响应**:
   ```
   收到需求，开始分析...

   分析完成！我将创建以下任务：

   1. task-001: 创建登录表单组件 (high)
   2. task-002: 实现登录 API 接口 (high)
   3. task-003: 添加表单验证逻辑 (normal)

   已创建状态报告：inbox/human_feedback/analysis-20260320-103000.md

   请确认任务优先级是否正确。
   ```

5. **用户确认**:
   在状态报告中回复：
   ```markdown
   ## 人类反馈

   - [x] 确认任务优先级正确
   - [x] 同意开始执行
   ```

6. **执行智能体响应**:
   ```
   已领取任务 task-001, task-002, task-003
   开始执行...

   创建分支：feature/task-001-login-form
   开发中...

   代码已提交：abc123
   已推送分支：feature/task-001-login-form

   已创建 PR #1
   已发送 Review 请求给 @tech-manager

   状态报告：inbox/human_feedback/dev-20260320-150000.md
   ```

7. **技术经理响应**:
   ```
   开始 Review PR #1...

   Review 完成：
   - 代码质量：✓
   - 测试覆盖：✓
   - 验收标准：✓

   PR 已合并到 main 分支
   任务 task-001 状态更新为：done
   ```

8. **最终总结**:
   ```
   所有任务完成！

   成果汇总:
   - 3 个任务完成
   - 3 个 PR 合并
   - 15 个文件新增/修改

   项目已就绪，可以开始下一阶段开发。
   ```
