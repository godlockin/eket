策略 4: 时间强制限制 ✅

实现: 通过 task-time-tracker.sh 和 config.yml 配置实现

强制限制类型: 1. 预估时间 - 任务领取时设置 2. 截止时间 - 基于预估时间自动计算 3. 心跳超时 - 30 分钟无响应触发重置 4. 总超时 - 180 分钟超时触发仲裁

触发条件: - 心跳超时 → 检查 Slaver 存活 → 重置状态或请求仲裁 - 总超时 → 检查 Slaver 存活 → 请求仲裁或重置状态

测试结果: 通过

策略 5: 同步检查和 Master 监控 ⚠️

实现文件: scripts/task-time-tracker.sh (master_monitor_loop)

功能:

master_monitor_loop() {
    while true; do
        for ticket_file in jira/tickets/*/*.md; do
            check_task_timeout "$ticket_id"
        done
        sleep 60  # 每分钟检查一次
    done
}

配置参数:

monitoring:
  master:
    check_interval_seconds: 60
  slaver:
    heartbeat_interval_seconds: 30
    timeout_seconds: 300
  task_timeout:
    check_interval_seconds: 120

状态: ⚠️ 脚本已创建，但需要 Master 实例启动时运行

策略 10: 任务完成验收机制 ⚠️

实现文件: scripts/merge-validator.sh

验证流程:

1. verify_feature_completion() - 检查验收标准
2. verify_unit_tests_pass() - 运行/检查 UT
3. verify_master_review_pass() - 检查 Review 状态
4. [全部通过] → 允许合并
5. [部分失败] → 创建后续任务 (FIX-XXX-post)

配置:

review:
  merge_strategy:
    all_features_complete: true
    unit_tests_pass: true
    master_review_pass: true
    incomplete_features:
      action: "create_follow_up_ticket"
      prefix: "FIX"

状态: ⚠️ 脚本已创建，待集成到 eket-review.sh / eket-merge.sh

建议: 在 PR 合并流程中调用验证脚本

策略 11: Checkpoint/Sprint/Retrospective ✅

实现文件: scripts/checkpoint-sprint-retro.sh

Checkpoint 类型: - task_start - 任务开始 (依赖检查、分支创建、计时器启动) - task_dev_complete - 开发完成 (UT 通过、Lint 通过) - task_test_complete - 测试完成 (E2E 通过、PR Draft) - task_complete - 任务完成 (功能验证、文档更新)

Sprint 管理: - start_sprint() - 启动 Sprint (7 天周期，5 个任务) - update_sprint_progress() - 更新进度 - complete_sprint() - 完成 Sprint

Retrospective: - 每 2 个 Sprint 后自动触发 - Stage 完成后触发 - 输出到 confluence/projects/*/retrospectives/

集成点: - ✅ eket-claim.sh - 领取任务时创建 task_start checkpoint

风险和已知问题

高风险

无 - 所有核心功能已实现并测试通过

中风险

  1. Master 监控未自动启动
    • 影响：任务超时不会自动检测
    • 解决：在 eket-start.sh Master 模式中添加 task-time-tracker.sh monitor
  2. Memory Review 未自动启动
    • 影响：记忆文件可能超过限制
    • 解决：在 Master 启动时添加 memory-review-agent.sh loop &
  3. Merge Validator 未集成
    • 影响：合并时不会自动验证
    • 解决：在 eket-merge.sh 中添加验证调用

低风险

  1. 脚本路径依赖 - 部分脚本使用相对路径，可能在某些环境下失败
  2. 日志目录不存在 - 首次运行时可能缺少日志目录
