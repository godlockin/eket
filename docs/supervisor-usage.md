# Supervisor 脚本使用文档

## 概述

`supervisor.sh` 是独立于 Claude/Node.js 的外部监控进程，负责监控 Master 和 Slaver 的心跳文件，检测假死并生成告警。

---

## 功能

### 1. Master 心跳监控
- **监控文件**: `.eket/state/master-heartbeat`
- **超时阈值**: 300 秒 (5 分钟)
- **告警行为**: 创建 `.eket/inbox/[CRITICAL]-MASTER-<timestamp>.md` 告警文件
- **告警内容**: 包含最后心跳时间、影响说明、恢复步骤

### 2. Slaver 心跳监控
- **监控文件**: `.eket/state/slaver-*-heartbeat`
- **超时阈值**: 650 秒 (~11 分钟)
- **记录行为**: 追加到 `.eket/state/recovery-queue.txt`
- **队列格式**: `<slaver_id>|<timestamp>`

### 3. 日志记录
- **日志文件**: `.eket/logs/supervisor.log`
- **日志级别**: DEBUG, INFO, WARN, ERROR, CRITICAL
- **日志格式**: `[ISO8601] [LEVEL] <message>`

## 启动方式

### 方式 1: 前台运行（测试）

```bash
bash scripts/supervisor.sh
```

**输出示例**:
```
[2026-05-14T23:44:34+08:00] [INFO] Supervisor started (PID: 53893)
[2026-05-14T23:44:34+08:00] [INFO] Project root: /Users/chenchen/.../eket
[2026-05-14T23:44:34+08:00] [INFO] Master timeout: 300s, Slaver timeout: 650s
[2026-05-14T23:44:34+08:00] [INFO] Check interval: 60s
[2026-05-14T23:44:34+08:00] [INFO] --- Heartbeat check cycle start ---
```

**终止**: `Ctrl+C` 或发送 `SIGTERM`

---

### 方式 2: 后台运行（推荐生产环境）

```bash
nohup bash scripts/supervisor.sh >> .eket/logs/supervisor.log 2>&1 &
echo $! > .eket/state/supervisor.pid
```

**查看进程**:
```bash
ps aux | grep supervisor.sh
# 或
cat .eket/state/supervisor.pid | xargs ps -p
```

**终止进程**:
```bash
cat .eket/state/supervisor.pid | xargs kill
```

---

### 方式 3: systemd (Linux)

创建 `/etc/systemd/system/eket-supervisor.service`:

```ini
[Unit]
Description=EKET Supervisor - Heartbeat Monitor
After=network.target

[Service]
Type=simple
WorkingDirectory=/path/to/eket
ExecStart=/bin/bash /path/to/eket/scripts/supervisor.sh
Restart=always
RestartSec=10
StandardOutput=append:/path/to/eket/.eket/logs/supervisor.log
StandardError=append:/path/to/eket/.eket/logs/supervisor.log

[Install]
WantedBy=multi-user.target
```

**启动**:
```bash
sudo systemctl daemon-reload
sudo systemctl enable eket-supervisor
sudo systemctl start eket-supervisor
sudo systemctl status eket-supervisor
```

---

### 方式 4: cron (定期启动，不推荐)

```bash
# 每 5 分钟检查一次（不推荐，因为无法保证连续监控）
*/5 * * * * cd /path/to/eket && bash scripts/supervisor.sh
```

**注意**: cron 方式无法实现 60 秒间隔的持续监控，仅适合低频检查。

## 配置

### 环境变量

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `EKET_ROOT` | 自动检测 | 项目根目录路径 |
| `SUPERVISOR_INTERVAL` | 60 | 检查间隔（秒） |

**自定义间隔示例**:
```bash
SUPERVISOR_INTERVAL=30 bash scripts/supervisor.sh  # 每 30 秒检查一次
```

### 修改超时阈值

编辑 `scripts/supervisor.sh`:

```bash
# Thresholds (seconds)
MASTER_TIMEOUT=300   # 修改为所需值
SLAVER_TIMEOUT=650   # 修改为所需值
```

## 告警文件示例

### Master 告警 (`[CRITICAL]-MASTER-1778773474.md`)

```markdown
# Master Heartbeat Timeout

**Time**: 2026-05-14T23:44:34+08:00
**Last Heartbeat**: 2026-05-14T23:36:43+08:00
**Status**: Master process appears dead (>300 s timeout)

## Symptom
Master heartbeat file has not been updated for more than 300 seconds.

## Impact
- No new tasks analyzed/assigned
- PR reviews blocked
- Slaver progress reports unprocessed

## Action Required
1. Check Master logs: `.eket/logs/master.log`
2. Verify Master Claude session running
3. Restart: `/eket-start -r master`
4. Check pending messages: `shared/message_queue/inbox/`

**Alert ID**: MASTER-1778773474
**Generated by**: supervisor.sh
```

### 恢复队列 (`recovery-queue.txt`)

```
001|1778773474
002|1778773500
```

**格式**: `<slaver_id>|<unix_timestamp>`

## 故障排查

### 1. 脚本无法启动

**症状**: `Error: Unsupported OS`

**原因**: 操作系统不支持

**解决**: 使用 macOS 或 Linux，或在 Windows 上使用 WSL

---

### 2. 无法生成告警文件

**症状**: 心跳超时但未生成 `[CRITICAL]-MASTER-*.md`

**排查**:
```bash
# 检查 inbox 目录权限
ls -ld .eket/inbox/

# 检查心跳文件是否存在
ls -la .eket/state/master-heartbeat

# 手动测试文件写入
echo "test" > .eket/inbox/test.md && rm .eket/inbox/test.md
```

---

### 3. 重复告警

**症状**: 同一次超时生成多个告警文件

**原因**: 多个 Supervisor 实例同时运行

**解决**:
```bash
# 查找所有 Supervisor 进程
ps aux | grep supervisor.sh

# 终止多余进程
kill <PID>
```

---

### 4. 恢复队列重复条目

**症状**: `recovery-queue.txt` 中同一 Slaver 多次出现

**原因**: Slaver 心跳文件长期未更新

**解决**:
1. 检查 Slaver 进程是否存活
2. 手动清理恢复队列:
   ```bash
   > .eket/state/recovery-queue.txt
   ```

---

## 性能影响

- **CPU**: < 1%（每分钟运行 2-3 秒）
- **内存**: < 10 MB
- **磁盘 I/O**: 极低（仅读取心跳文件 mtime）

---

## 安全注意事项

1. **文件权限**: 确保 `.eket/inbox/` 可写，但限制其他用户访问
2. **日志轮转**: 定期清理 `.eket/logs/supervisor.log`（可用 `logrotate`）
3. **PID 文件**: 防止多实例运行冲突
