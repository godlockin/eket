Basic Usage

import { createTaskAssigner } from 'eket-cli/core';
import { claimTask } from 'eket-cli/commands';

// Assign task
const assigner = await createTaskAssigner(config);
const result = await assigner.assign({
  ticketId: 'TASK-001',
  requiredRole: 'backend_dev'
});

// Claim task (CLI)
await claimTask('TASK-001', { role: 'backend_dev' });

📦 Core Modules Overview

Core Module

Agent & Task Management

  • AgentPool - Agent instance pool with health checks and load balancing
  • TaskAssigner - Intelligent task assignment with role matching
  • InstanceRegistry - Agent registration and status tracking
  • EnvelopeManager - Message passing infrastructure
  • WorktreeManager - Git worktree isolation for parallel development
  • SagaExecutor - Distributed transaction management with automatic rollback

Key Features: - ✅ Redis-backed distributed state - ✅ SQLite persistence for metadata - ✅ Automatic health monitoring - ✅ Load balancing (round-robin / least-tasks) - ✅ SSE event bus integration

Commands Module

CLI Interface for EKET Operations

  • task:claim - Claim Jira tasks with role matching
  • task:complete - Complete tasks and submit PR with validation
  • task:status - View agent status and available tasks
  • knowledge:search - Full-text search in knowledge base
  • knowledge:index - Rebuild search index with proof validation
  • deploy:sync-branches - Three-branch sync (feature → testing → main → miao)
  • agent:poll - Background polling for tasks/messages (Master/Slaver)
  • alerts:list - View and manage system alerts

Key Features: - ✅ Interactive CLI with ora spinners - ✅ Nyquist Rule enforcement (automated acceptance criteria) - ✅ Execution Proof validation for knowledge base - ✅ Saga-based transaction management - ✅ SSE event notifications

🎯 Common Use Cases

Use Case 1: Claim and Complete a Task

# 1. Claim task
eket task:claim TASK-001 --role backend_dev

# 2. Develop in isolated worktree
cd .eket/worktrees/TASK-001
# ... coding ...

# 3. Run acceptance tests
npm test -- --testPathPattern=auth

# 4. Complete task (auto PR creation)
eket task:complete TASK-001

Use Case 2: Search Knowledge Base Before Development

# Search for similar patterns
eket knowledge:search "API documentation"

# Check for pitfalls
eket knowledge:search --type pitfall "TypeDoc"

Use Case 3: Deploy to Production

# Sync branches
eket deploy:sync-branches feature/TASK-001

# Auto-merge to testing → main → miao
# Conflicts resolved automatically (configurable strategy)

Use Case 4: Monitor Agent Health

import { createAgentPool } from 'eket-cli/core';

const pool = await createAgentPool(config);

// Run health check
const unhealthy = await pool.healthCheck();

if (unhealthy.length > 0) {
  console.warn('Unhealthy agents:', unhealthy);
  await pool.removeAgents(unhealthy);
}

📖 TypeScript API Examples

Example 1: Task Assignment

import { createTaskAssigner } from 'eket-cli/core';

const assigner = await createTaskAssigner({
  projectRoot: '/path/to/project',
  redis: { enabled: true, host: 'localhost' },
  sqlite: { path: '.eket/state/eket.db' }
});

const result = await assigner.assign({
  ticketId: 'TASK-001',
  requiredRole: 'backend_dev',
  strategy: 'round_robin'
});

if (result.success) {
  console.log(`Task assigned to: ${result.assignedTo}`);
} else {
  console.error(`Assignment failed: ${result.reason}`);
}

Example 2: Message Passing

import { EnvelopeManager } from 'eket-cli/core';

const manager = new EnvelopeManager('/project/root');

// Send message
await manager.send({
  type: 'task_claim',
  from: 'slaver-001',
  to: 'master',
  payload: { ticketId: 'TASK-001' },
  priority: 'high'
});

// Receive messages
const messages = await manager.receive('master', { limit: 10 });

for (const msg of messages) {
  console.log(`[${msg.type}] from ${msg.from}:`, msg.payload);
}

Example 3: Error Handling

import { printError, logSuccess } from 'eket-cli/utils';

try {
  await riskyOperation();
  logSuccess('Operation completed successfully');
} catch (error) {
  printError({
    code: 'OPERATION_FAILED',
    message: 'Failed to complete operation',
    details: { error: error.message },
    causes: ['Network timeout', 'Invalid configuration'],
    solutions: [
      'Check network connection',
      'Validate config.yml with: eket config:validate'
    ],
    quickFix: 'eket config:validate',
    docLink: 'https://eket.dev/docs/troubleshooting',
    severity: 'error'
  });
  process.exit(1);
}

🐛 Troubleshooting

Common Errors

Error Cause Solution
REDIS_UNAVAILABLE Redis not running redis-server &
SQLITE_LOCKED Database locked by another process Close other EKET instances
NO_AVAILABLE_AGENT No agents with matching role Register more agents or adjust role filter
WORKTREE_CONFLICT Branch name already exists Use git worktree list to check
